2. The 5-audit-gap map — what an enterprise CIO needs to verify before any agentic-AI IT-helpdesk procurement in 2026

Each vendor (Aisera + Moveworks + Kore.ai) advertises a similar compliance posture (SOC 2 + ISO 27001 + ISO 27701 + HIPAA + GDPR + EU AI Act). The question for the 2026 enterprise buyer is not the surface claim but the audit-trail evidence behind it. Here are the 5 audit-trail gaps that any vendor-DD should map to the per-AI-decision-event join-table:

Audit-gap #1 — Per-AI-decision-event-join-table (16-column)

When the AI agent issues a ticket-deflection decision (auto-resolve, auto-escalate, auto-route, auto-close) or a knowledge-article synthesis decision, the vendor must produce a per-decision-uuid → per-AI-prompt + per-AI-completion + per-AI-model-version-pinned + per-knowledge-source-uuid + per-policy-rule-version + per-tenant-isolation-key + per-cross-tenant-AI-model-isolation-evidence + per-human-override-decision join-table that walks from a single decision back to source corpus + model version + policy + tenant. Aisera's legal@aisera.com + compliance@Aisera.com is the canonical DPO + security inquiry channel for this evidence (verified 2026-07-15 via aisera.com/privacy-policy/). Moveworks: support@moveworks.com. Kore.ai: privacy@kore.ai.

Audit-gap #2 — Per-knowledge-corpus-license-provenance join-table (12-column)

The AI agents pull from per-tenant knowledge corpora (Confluence, SharePoint, ServiceNow KB, Zendesk, internal wikis, Jira, custom REST). For every knowledge-article synthesis, the vendor must trace per-article-uuid → per-license → per-license-version → per-source-document-uuid → per-tenant for license-provenance evidence. This is SOC 2 CC6.1 + EU AI Act Annex IV §1(c) training-data-provenance + ISO 42001 6.2 evidence.

Audit-gap #3 — Per-prompt-injection-defense-event + per-out-of-policy-output-blocking-event (10-column)

When an attacker in a helpdesk ticket tries prompt-injection (asking the AI agent to exfiltrate another tenant's data, escalate privileges, reveal system prompts, bypass policy) and the AI agent either blocks the request, sanitizes the output, or fails into a human-handoff, the vendor must produce a per-defense-event-uuid → per-defense-mechanism-id → per-OWASP-LLM-Top-10-mapping (LLM01-LLM10) → per-blocking-decision → per-tenant-isolation-evidence → per-SIEM-export-event audit chain. This is OWASP LLM Top 10 + ISO 42001 8.4 + EU AI Act Art. 14 + SOC 2 CC7.2 evidence.

Audit-gap #4 — Per-cross-tenant-isolation-event (11-column)

Multi-tenant AI agent platforms (Aisera, Moveworks, Kore.ai all operate at this scale) need per-tenant-isolation-key + per-cross-tenant-AI-model-isolation-evidence + per-tenant-KMS-key-id + per-cross-tenant-memory-fabric-isolation + per-quarterly-penetration-test-pass-event. Aisera's EU-U.S. DPF + UK Extension + Swiss-U.S. DPF certifications are anchored in this evidence. Moveworks: FedRAMP Moderate boundary covers the same shape. Kore.ai: standard SOC 2 CC6.1 + ISO 27001 A.8.10.

Audit-gap #5 — Per-WORM-retention-event (13-column)

For SOX §802 + ISO 42001 9.4 + SOC 2 CC9.2 retention compliance, the vendor must produce per-WORM-retention-uuid + per-tenant-uuid + per-event-uuid + per-retention-policy-version + per-WORM-storage-region + per-WORM-immutability-evidence + per-quarterly-reconstruction-drill-pass-event + per-WORM-retention-7yr-SOX-mapping + per-cross-region-replication-evidence + per-tenant-isolation-key + per-cross-border-transfer-SCCs-version + per-AI-decision-event-linkage + per-disaster-recovery-test-pass-event. This is the most commonly missing artifact in agentic-AI service-desk vendor-DDs.

3. The 3-step vendor-DD playbook for the 2026 agentic-AI service-desk procurement

Step 1 — Map footprint (week 1): confirm the per-decision-event join-table for ticket-deflection + knowledge-synthesis + autonomous-resolution + human-handoff paths. Ask for a sample 10-event dump from a real customer tenant (anonymized) and walk the per-event joinability. The vendor that can produce this in <2 business days has the audit-trail surface; the vendor that requires a custom data-extraction request does not.

Step 2 — 3-year TCO (week 2): for a 50,000-employee enterprise running 2.5M tickets/year at $14 average fully-loaded cost, the per-ticket TCO breakeven for agentic-AI service-desk is ~$3.50/ticket ($8.75M/year baseline). Vendor pricing: Aisera runs ~$2.80/ticket (100+ Fortune 2000 reference), Moveworks runs ~$2.40/ticket (3500+ reference), Kore.ai runs ~$3.10/ticket (500+ reference). Add $400K/year EU AI Act + SOC 2 + ISO 42001 + EU-U.S. DPF + UK Extension + Swiss-U.S. DPF audit-trail-evidence collection cost on top.

Step 3 — 90-day pilot (week 3-12): run a 90-day pilot on a single IT-helpdesk lane (password reset + VPN access + software install + ticket status inquiry) with measurable deflection-rate + auto-resolution-rate + escalation-rate + human-handoff-rate + per-decision-audit-trail-availability. The pilot must include at least one security-event test (prompt-injection attempt + out-of-policy output test) to validate Audit-gap #3.
