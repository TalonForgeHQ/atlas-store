Laminar is an open-source, agent-first observability platform designed to trace long-running agents, replay a failed execution from a chosen step, and detect failure patterns at scale. That execution-reconstruction surface is a practical starting point for EU AI Act logging, human-oversight, and robustness evidence.

Why Laminar is different from request-response LLM observability

Generic LLM monitoring often stops at prompt, completion, token count, latency, and cost. Long-running agents fail across a chain: an earlier state mutation changes a later retrieval, a tool call returns a partial result, or a retry runs under a different model or deployment version. Laminar's trace-and-replay model can preserve the causal chain needed to reproduce the failure.

  • Trace: capture the complete agent run rather than one isolated model call.
  • Step: locate the exact agent step where behavior diverged.
  • State snapshot: record the minimum state needed to reproduce the next action.
  • Replay origin: restart analysis from the failed step instead of rerunning the entire workflow.
  • Run delta: compare the original failure with the corrected replay.
  • Pattern detection: group recurring failures across production traces.

EU AI Act Article 12: automatic logging and reconstruction

For a deployer or provider operating an in-scope high-risk system, the useful question is not "do traces exist?" It is "can a reviewer reconstruct the material event without searching five disconnected tools?" A Laminar evidence map should join the agent run, failed step, model and prompt version, tool call, state hash, replay, reviewer decision, and deployment version into one exportable record.

Article 14: human-oversight receipts

  • Record who initiated the replay and why.
  • Record whether the correction was approved, rejected, or escalated.
  • Record the rollback or containment action when the replay still fails.
  • Record acknowledgment time for a production incident.
  • Separate diagnostic access from production-release authority.

Article 15: robustness evidence from replay deltas

The strongest robustness packet compares an original run with a replay under controlled changes. The report should identify the changed prompt, model, tool policy, state transition, or deployment version; show the evaluation result; and link the accepted correction to a regression test and production release.

Managed cloud and self-hosted evidence boundary

Laminar is open source and supports self-hosting, so the audit should distinguish product capability from each customer's deployment responsibility. The evidence map should state who controls encryption keys, regions, retention, backups, tenant isolation, deletion, sub-processors, and incident response in managed-cloud and self-hosted configurations.
