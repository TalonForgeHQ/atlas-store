5-audit-gap map (where the audit-trail typically fails)

  1. No per-AI-Decisioning-suggestion provenance — the LLM-suggested personalization or audience update is logged as a single sync-row, not a separate per-suggestion event with prompt + retrieved-segment + model-id + human-override — which means auditors cannot trace the AI's reasoning chain back to a specific prompt-template + retrieved-segment.
  2. No per-destination-API-call attribution — Hightouch logs sync-level success/fail but not per-API-call, so when an audit asks "which Salesforce record was written, with which PII fields, at which UTC timestamp, with which actor-id" the answer is "we know a sync ran."
  3. No third-party-agent-write join key — when an external agent (Atlas or otherwise) writes back to the same destination Hightouch just wrote to, there is no shared key — auditors cannot reconstruct which upstream Hightouch sync informed the downstream agent decision.
  4. Per-tenant KMS key + BYOK not surfaced — auditors want to verify per-tenant-key-id + customer-managed-key (CMK) + bring-your-own-key (BYOK) + WORM retention + cross-border-transfer-SCCs-version in a single joinable surface; most reverse-ETL vendors surface these via a security questionnaire, not as a joinable audit-trail event.
  5. No per-prompt-injection defense event — when a downstream SaaS destination returns a tool-call response that contains adversarial instructions, most reverse-ETL AI layers pass it through; a defensible Hightouch audit-trail should surface per-prompt-injection-defense-event joinable per-AI-Decisioning-suggestion-id.
