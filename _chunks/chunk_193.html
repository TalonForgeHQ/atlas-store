1. The 26 Cognigy-specific audit-gaps (grouped by evidence surface)

1.1 Per-conversation-flow-graph (6 audit-gaps)

  1. Per-conversation-id persistence — must be unique + immutable + cross-system-joined (flow → intent → entity → context → LLM → NER → translation → sentiment → handover → AI action → resolution).
  2. Per-flow-id + per-flow-version — Cognigy ships flow-version-control as a first-class export surface. Must be Git-style with 12-state change-history (draft → review → approved → active → deprecated → archived).
  3. Per-flow-step-id — every node in the conversation flow graph gets a stable step-id. Auditor wants the exact step-type (intent-classification, entity-extraction, slot-filling, API-call, LLM-prompt, NER-lookup, sentiment-score, translation-lookup, handover-decision, action-execution) and step-transition.
  4. Per-intent-id + per-intent-version — Cognigy's intent classifier is the foundation of every conversational decision. Auditor wants intent-version-control + intent-confidence-threshold + intent-fallback-policy.
  5. Per-entity-id + per-slot-id — Cognigy's NER engine extracts entities (account-number, order-id, claim-number, vehicle-VIN, policy-number) into slots. Auditor wants per-slot schema-version + per-slot confidence-score + per-slot source-model.
  6. Per-context-window-id — Cognigy maintains a per-conversation context window. Auditor wants context-window TTL + context-window-encryption-at-rest + context-window PII-redaction-policy.

1.2 Per-LLM-and-NER-model-lineage (5 audit-gaps)

  1. Per-LLM-model-id + per-LLM-prompt-hash — which model answered (Claude Sonnet, GPT-4o, Cognigy fine-tuned, customer fine-tuned) + which prompt-template (Cognigy ships prompt-template-versioning as a Cognigy Copilot primitive).
  2. Per-prompt-template-version — Cognigy's prompt-template library is a first-class export surface. Must include template-hash + template-owner + template-review-status + template-effective-date.
  3. Per-prompt-injection-detection-signal-id — Cognigy ships prompt-injection-detection as a runtime guard. Auditor wants per-injection-attempt-attribution + per-injection-mitigation-action (block, sanitize, escalate-to-human, log-and-continue) + per-injection-false-positive-rate-tracking.
  4. Per-NER-model-id + per-NER-confidence-score — which NER model extracted which entity (German, French, Spanish, English, customer-specific fine-tuned). Auditor wants per-language NER-model-version + per-language NER-accuracy-benchmark.
  5. Per-translation-model-id — which translation model handled which message (DeepL, Google Translate, customer-trained). Auditor wants per-language translation-accuracy + per-message source-language-detection-confidence + per-message translation-cache-hit-rate.

1.3 Per-sentiment-and-human-handover (5 audit-gaps)

  1. Per-sentiment-score — Cognigy's sentiment classifier runs on every message. Auditor wants per-message sentiment-score + per-conversation sentiment-trajectory + per-sentiment-threshold escalation-trigger.
  2. Per-handover-id — every AI-to-human handoff gets a stable handover-id. Auditor wants handover-trigger-reason + handover-context-snapshot-exported-to-human-agent + handover-time-to-first-human-response.
  3. Per-handover-tier — Cognigy supports multi-tier handover (Tier-1 support, Tier-2 specialist, supervisor, escalation-team). Auditor wants per-tier escalation-policy + per-tier SLA + per-tier human-agent-skill-match.
  4. Per-human-agent-id — every human responder in a conversation gets attributed. This is the regulated-accountability surface: Lufthansa wants the actual gate-agent's employee-id, Bosch wants the actual field-technician's badge-id, Mercedes-Benz wants the actual dealer-service-advisor's associate-id. Auditor wants per-human-agent skill-routing + per-human-agent conversation-ownership + per-human-agent regulatory-licensing-attestation.
  5. Per-AI-action-id + per-action-type + per-action-rollback-id — Cognigy's AI agents execute actions (refund, account-update, ticket-create, callback-schedule, shipment-reroute). Auditor wants per-action rollback-window + per-action rollback-reason + per-action reversal-attestation.

1.4 Per-resolution-quality-and-rollback (5 audit-gaps)

  1. Per-resolution-quality-score — Cognigy ships conversation-quality-scoring as a Cognigy Insights primitive. Auditor wants per-conversation quality-score + per-conversation CSAT-NPS-correlation + per-conversation quality-drift-detection.
  2. Per-conversation PII-redaction-policy-id — Cognigy redacts PII (PCI card, SSN, IBAN, account-number) before persisting conversation logs. Auditor wants per-redaction-rule version + per-redaction false-negative-rate-tracking + per-redaction audit-log.
  3. Per-conversation retention-policy-id — GDPR Art. 5(1)(e) + CCPA/CPRA require explicit retention windows. Cognigy supports per-conversation TTL + per-conversation legal-hold-export + per-conversation right-to-erasure execution.
  4. Per-conversation WORM-export-target — Cognigy exports conversation logs to immutable WORM (write-once-read-many) targets (AWS S3 Object Lock, Azure Blob Immutable Storage, customer on-prem WORM). Auditor wants per-export WORM-retention-period + per-export WORM-legal-hold-flag + per-export WORM-chain-of-custody.
  5. Per-conversation cross-system-join-id — the conversation-id must join to the customer's CRM (Salesforce, ServiceNow, Zendesk), ERP (SAP, Oracle), and identity-provider (Okta, Azure AD) records. Auditor wants per-conversation CRM-trail + per-conversation ERP-trail + per-conversation IdP-trail.

1.5 Per-conversation-context-window-attestation (5 audit-gaps)

  1. Per-context-window encryption-key-id — the context-window encryption key must be customer-managed (BYOK / HYOK). Auditor wants per-key rotation-cadence + per-key HSM-attestation + per-key access-log.
  2. Per-context-window PII-detection-id — Cognigy's PII-detector runs on every message before persistence. Auditor wants per-detection rule-version + per-detection false-negative-rate + per-detection audit-log.
  3. Per-context-window multi-language-policy-id — Cognigy supports 100+ languages. Auditor wants per-language PII-detection-rule + per-language translation-accuracy-SLA + per-language data-residency.
  4. Per-conversation data-residency-region — Cognigy deploys in EU (Frankfurt), US (Virginia), APAC (Singapore), customer on-prem. Auditor wants per-conversation region-attestation + per-region data-sovereignty-compliance + per-region GDPR/Schrems-II / APAC-Cross-Border-Privacy-assessment.
  5. Per-conversation human-review-flag — Cognigy supports human-review-flag (sample-and-review, full-review, escalation-required). Auditor wants per-conversation human-review-trigger + per-conversation human-review-outcome + per-conversation human-review-corrective-action.
  1. Per-conversation prompt-injection-mitigation-outcome — when the prompt-injection-detection fires, Cognigy executes one of: block, sanitize, escalate-to-human, log-and-continue. Auditor wants per-mitigation outcome + per-mitigation false-positive-rate + per-mitigation appeal-process.
