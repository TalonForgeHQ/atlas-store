1. The 24 Spellbook-specific audit-gaps (grouped by evidence surface)

1.1 Per-clause-version-id + per-redline-suggestion-id + per-clause-accept-reject-event drafting-provenance (5 audit-gaps)

  1. Per-clause-id + per-clause-version-id + per-parent-contract-id — every clause in every Spellbook-assisted draft must have a unique, immutable, cross-system-joined ID (Spellbook → Word/Google Docs export → DocuSign → iManage/NetDocuments DMS → client-matter-management system). The audit trail needs clause-id, clause-version-id, parent-contract-id, clause-position (section + paragraph + sub-paragraph), and the contract-milestone timestamp (draft-1, draft-2, partner-review, client-circulation, executed).
  2. Per-redline-suggestion-id + per-redline-suggestion-accept-reject-event + per-redline-suggestion-confidence-score — every AI-suggested redline (clause-insert, clause-replace, clause-delete, clause-rewrite) needs a redline-suggestion-id that joins to the clause-id + the LLM-model-id + the prompt-template-version-id + the human-reviewer-id + the accept/reject decision. The audit trail needs the suggestion-confidence-score (Spellbook's redline-confidence) + the source-clause-id + the destination-clause-id.
  3. Per-clause-risk-flag-id + per-clause-risk-flag-tier (Tier-0 non-standard / Tier-1 yellow / Tier-2 red / Tier-3 deal-breaker) — Spellbook's risk-flag system must export a structured risk-flag-id + risk-flag-tier + risk-flag-reason + risk-flag-disposition (accepted-by-partner / accepted-by-GC / rejected-and-revised / escalated-to-insurance-carrier). Each tier triggers a different ABA + state-bar + AmLaw-200 supervisory-attorney disclosure obligation.
  4. Per-precedent-repository-id + per-precedent-clause-version-id — when Spellbook cites a precedent clause (from the firm's precedent repository, from the firm's prior deal documents, from the firm's playbook), the audit trail needs the precedent-repository-id + the precedent-clause-id + the precedent-clause-version-id + the citation-context-id. The auditor needs to verify the precedent was sourced from a firm-controlled repository, not from a public LLM training set that may contain client-confidential information.
  5. Per-matter-id + per-engagement-id + per-billing-partner-id + per-client-matter-id — every Spellbook-assisted draft must be joined to the matter-id + the engagement-letter-id + the billing-partner-id + the ABA-required client-matter-conflict-check-id. The audit trail needs to demonstrate the AI-assisted draft was performed within the scope of the engagement letter and that the conflict-check was performed before any client-confidential information was ingested into Spellbook.

  2. Per-completion-hash + per-completion-token-count + per-completion-cost-usd — every Spellbook completion (redline-suggestion, clause-draft, risk-flag-explanation) needs a completion-hash that joins to the model-id + the prompt-template-version-id + the downstream-clause-version-id. The audit trail needs the completion-hash to be cryptographically verifiable so the auditor can confirm the completion was not retroactively altered.
  3. Per-grounded-citation-id + per-grounded-source-clause-id + per-grounded-precedent-id — every grounded redline-suggestion needs a grounding-evidence-id that ties the suggestion to the source-clause(s) it came from. The audit trail needs the grounding-confidence-score + the source-clause-ids + the LLM-model-id that produced the suggestion.
  4. Per-prompt-template-rollback-event + per-prompt-template-rollback-reason + per-prompt-template-rollback-human-reviewer-id — when Spellbook rolls back a prompt-template-version (e.g. due to a hallucinated redline-suggestion that escaped to a client), the rollback event must include the original prompt-template-version + the policy decision + the connector permission scope + the human-reviewer-id + the compensating action ID. 1.3 Per-AI-action-override-id + per-action-rollback-id + per-action-rollback-reason write-action-lineage (4 audit-gaps)

  1. Per-AI-action-id + per-action-type (clause-insert / clause-replace / clause-delete / clause-rewrite / risk-flag-set / precedent-insert) — when the Spellbook agent makes a write into the draft (clause-insert, clause-replace, clause-delete, clause-rewrite, risk-flag-set, precedent-insert), the action-id needs to be unique + immutable + cross-system-joined (Spellbook → Word/Google Docs → DocuSign → iManage/NetDocuments → client-matter-management). The audit trail needs the action-type + the action-target-clause-id + the action-timestamp.
  2. Per-AI-action-override-id + per-override-reason + per-override-human-reviewer-id — when the supervising attorney + the partner + the GC overrides an AI action, the override-id + override-reason + override-human-reviewer-id must be in the same row as the action-id. ABA Model Rule 5.1 + 5.3 require this evidence to demonstrate the attorney exercised reasonable supervisory oversight of the nonlawyer assistance.
  3. Per-action-rollback-id + per-action-rollback-reason + per-action-rollback-human-reviewer-id — when the supervising attorney asks Spellbook to "undo what the AI did" (e.g. a hallucinated risk-flag set on a non-standard but firm-approved clause), the rollback artifact must include the original prompt + the policy decision + the connector permission scope + the human-reviewer-id + the compensating action ID.
  4.
