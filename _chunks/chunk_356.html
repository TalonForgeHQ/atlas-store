The 5 audit gaps we map (the production-chain reconstruction lens)

The Sembly AI production chain is exactly the chain meeting → recording → transcript → summary / digest → analytics → report → downstream delivery. Each step is an evidence link a buyer (or auditor) has to be able to reconstruct end to end. The five canonical audit gaps are:

Gap 1 — Meeting-to-report provenance reconstruction

Can Sembly reconstruct one meeting report end to end from a single meeting ID? The chain to verify: meeting ID → consent / recording event → transcript span → speaker attribution → retrieved context → prompt / model version → summary or digest → analytic signal → final report → downstream delivery recipient. If any link in that chain is opaque, the enterprise buyer cannot answer the question "show me everything that influenced this report" — and that question is the spine of every SOC 2 CC7.2 / GDPR Art. 30 / EU AI Act Art. 12 logging / ISO 42001 evidence review.

Gap 2 — Consent, access, and deletion propagation

Buyers need traceable controls across all of: recordings, transcripts, summaries, digests, analytics, reports, search indexes, integrations, backups, and subprocessors. The Privacy Policy explicitly enumerates many of these surfaces, which is a strong posture signal — but the propagation evidence (a deletion request under GDPR Art. 17 against the meeting ID → observable tombstone in the transcript index, the summary cache, the analytics warehouse, the search shard, and the subprocessor replication paths) is the audit question that buyers and regulators actually ask. Map it to SOC 2 CC6.1 + CC7.2 + GDPR Art. 5(1)(e) + Art. 17 + ISO 27701 PIMS B.8.4.

Gap 3 — Hostile meeting-input defense

Spoken instructions, chat, calendar text, and shared documents inside a meeting can carry prompt injection or poisoned context. The OWASP LLM Top 10 LLM01 (prompt injection) + LLM03 (training-data poisoning) + LLM06 (sensitive disclosure) + LLM08 (vector and embedding weaknesses) + MITRE ATLAS AML.T0048 (erosion of ML model integrity) all apply. The audit question: what detection and human-review evidence travels with the affected output? Map to OWASP LLM Top 10 + MITRE ATLAS + EU AI Act Art. 14 human-oversight + ISO/IEC 42001 A.6.3.5.

Gap 4 — Tenant, plan, and residency isolation

Customer content, model-training settings, subprocessors, EU-only storage toggle, AI outputs, and access permissions must remain isolated across workspaces, plans, and regions. The Privacy Policy confirms EU-only data storage is available as an option — that is the right product move — but the implementation evidence (which data plane + which subprocessor region + which KMS key + which retention job + which analytics warehouse carries the EU-only bit) is the audit question. Map to SOC 2 CC6.1 + GDPR Art. 28 + Schrems II SCC + EU-US DPF + ISO 27001 A.8.12 + ISO 27701 B.8.2.3.

Gap 5 — Security and finance reconciliation

Immutable evidence + per-meeting attribution for capture, transcription, model use, storage, analytics, subprocessor cost, and downstream delivery lets security, compliance, and finance reconcile the same run. This is the gap that turns audit-response time from weeks into hours and lets margin review ship inside the same week as the engagement. Map to SOC 2 CC7.2 + ISO 42001 A.6.3.6 + the buyer's own finance-grade cost-attribution drill.
