The 5 procurement-evidence vectors

Vector 1 — Per-tenant scoping + cross-tenant-no-bleed proof

Every Fortune-500 buyer asks: "show me a per-tenant boundary that proves customer-A traces cannot be read by customer-B at the database or API layer." Langfuse ships this via per-tenant_id scoping across all 18 columns. Arize AI ships it via Phoenix OSS's per-project boundary + Arize AX's enterprise multi-tenant controls. Honeycomb inherits it from the APM side. Maxim AI + LangSmith ship it via per-workspace scoping.

Vector 2 — Per-prompt-template + per-prompt-version lineage

The procurement question: "if I pin a prompt template at version 47, can you prove every LLM call that ran with that version across the eval period?" Langfuse ships llm_prompt_template_id + llm_prompt_template_version_id natively. Arize AX ships it via the prompt-versioning surface. Honeycomb maps it via custom trace attributes. Maxim AI + LangSmith ship it via prompt-registry surfaces.

Vector 3 — Per-LLM-judge versioning + per-eval-run reproducibility

The procurement question: "if you used GPT-4o as the LLM-judge on March 14, can you replay that exact eval-run today with the same model version + same prompt + same scoring rubric?" This is the eval-lineage vector. Langfuse + Arize both pin llm_eval_judge_id + llm_evaluator_llm_model_version_id. Honeycomb ships it via per-eval custom tags. Maxim AI + LangSmith ship it via per-eval-run reproducibility surfaces.

Vector 4 — EU AI Act + NIST AI RMF evidence-pinning per LLM-use-case

The procurement question: "for each high-risk LLM-use-case in our inventory, show me the eval evidence that maps to EU AI Act Article 9 (risk-management) + Article 13 (transparency) + Article 14 (human oversight) + Annex III high-risk list." This is the regulatory-evidence vector — every vendor will need to ship it by Q4 2026. Langfuse + Arize are furthest along here because they treat per-eval-run as first-class data.

Vector 5 — Sub-processor cascade + per-sub-processor change-notification SLA

The procurement question: "when you add a new sub-processor (say, a new LLM provider or a new vector DB), what is the customer-notification SLA?" Langfuse ships it via /pricing footer sub-processor list. Arize AX ships it via the enterprise DPA. Honeycomb ships it via the standard DPA. Maxim + LangSmith ship it via the standard enterprise agreement.

FAQ

Which LLM-observability vendor has the strongest open-source pedigree?

Arize AI's Phoenix OSS (github.com/Arize-ai/phoenix, MIT license) is the only cohort member with a first-party open-source trace collector that is also the recommended dev-side on-ramp to the commercial product. Langfuse is also MIT-licensed but ships the entire platform open-source (not just the trace collector). Honeycomb, Maxim, LangSmith are commercial-only.

Which vendor is best for multi-modal eval (text + image + structured)?

Arize AI (Arize AX ML & CV). Their eval surface natively handles text + image + structured-output LLM applications — strongest in cohort for teams shipping non-text-LLM products.

Which vendor has the cleanest EU AI Act + NIST AI RMF evidence surface?

Langfuse + Arize AI are tied here — both ship per-eval-run as first-class data with per-eval-run JSON export, which maps cleanly to EU AI Act Article 9 + 13 + 14 + Annex III evidence-pinning requirements. Maxim + LangSmith are close behind but ship less out-of-the-box.

Which vendor wins the multi-step agent-eval use case?

Maxim AI. Their agent-simulation + agent-trace-tree eval-runner is the cleanest in cohort for multi-step agent products. Arize AI + Langfuse ship single-LLM-call eval surfaces; Maxim ships the full agent-trace tree.

Which vendor wins the LangChain/LangGraph standardized team?

LangSmith. Same vendor as the framework, same license boundary, same release cadence — zero-config trace export from LangChain + LangGraph runtime.
