1. End-to-end provenance join-table per LLM call

The first question is the one that breaks every current AI-agent audit framework: how do you produce a single 18-column join-table per LLM call that links every artifact in the Datadog observability surface?

The reference join-table looks like this — and the auditor will want all 18 columns populated, immutable, with 14-month retention (Datadog's public Audit Trail retention floor):

CREATE TABLE datadog_ai_call_provenance (
  call_id                   UUID PRIMARY KEY,
  llm_call_span_id          VARCHAR(64),
  ai_agent_reasoning_chain  TEXT,
  davis_ai_nl2mon_query     TEXT,
  bits_ai_incident_action   TEXT,
  cloud_siem_ai_detection   VARCHAR(64),
  watchdog_auto_detection   VARCHAR(64),
  ai_guardrail_trigger      VARCHAR(64),
  llm_prompt_capture        TEXT,
  llm_completion_capture    TEXT,
  sensitive_data_redaction  JSONB,
  ml_model_bias_flag        BOOLEAN,
  ml_feature_store_mutation VARCHAR(64),
  otel_ai_inference_span    VARCHAR(64),
  otel_ai_tool_call_span    VARCHAR(64),
  otel_multi_agent_span     VARCHAR(64),
  audit_trail_event_id      VARCHAR(64),
  downstream_state_change   JSONB,
  ts                        TIMESTAMPTZ NOT NULL,
  retention_until           TIMESTAMPTZ NOT NULL  -- immutable 14-month
);

SOC 2 CC7.2 + EU AI Act Art. 12 (logging) + ISO 42001 §9.4 all require per-event provenance with a queryable join-table. Datadog's Audit Trail product is the surface that holds this — but only if every span (LLM + Agent + Davis + Bits + SIEM + Watchdog + Guardrail + ML) flows through it.
