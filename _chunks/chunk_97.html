The 16-column reasoning-chain provenance join-table auditors will demand

For SOC 2 CC7.2 + EU AI Act Art. 12 + ISO 42001 9.4, your auditor will expect a per-LLM-call provenance join-table with these 16 columns:

  1. per_llm_call_uuid — UUIDv7 for the LLM call (one row per Galileo Observe trace event)
  2. per_tenant_id — tenant boundary for cross-tenant isolation evidence (SOC 2 CC6.1 + GDPR Art. 28 + HIPAA + EU AI Act Art. 10 + FedRAMP Moderate)
  3. per_event_uuid — UUIDv7 for the Galileo Observe event (L1 provenance capture)
  4. per_trace_id — OpenTelemetry-compatible trace id linking the LLM call to the surrounding agent-step graph
  5. per_ai_decision_id — UUID for the AI-decision event (Galileo Protect gate decision + Galileo Agent Observability agent-decision + Galileo Evaluate metric-trigger)
  6. per_prompt_text — exact prompt text sent to the model (WORM-locked, hashed for storage, retrievable for SOC 2 CC7.2 + EU AI Act Art. 12 audit replay)
  7. per_completion_text — exact completion text returned by the model
  8. per_llm_token_cost — token cost of the LLM call in USD (input-tokens * input-price + output-tokens * output-price)
  9. per_tool_call_id — UUID for any tool-call invoked by the agent (linked to per-tool-call-decision-chain in Galileo Agent Observability)
  10. per_rag_retrieval_id — UUID for the RAG retrieval that fed the LLM prompt (linked to per-RAG-context-precision-recall)
  11. per_rag_context_precision — Galileo Evaluate metric for the fraction of retrieved context that was actually relevant
  12. per_rag_context_recall — Galileo Evaluate metric for the fraction of ground-truth context that was retrieved
  13. per_citation_groundedness — Galileo Evaluate metric for whether each cited claim is supported by the retrieved context
  14. per_hallucination_flag — Galileo Evaluate boolean flag (true/false) for whether the completion was classified as a hallucination
  15. per_agent_decision_id — UUID for the agent-decision event (Galileo Agent Observability)
  16. per_downstream_state_change_id — UUID for any downstream state change caused by the agent's action (linked to the auditor's reverse-join from Salesforce + HubSpot + Marketo + Segment + mParticle downstream-state-change back to the Galileo AI-call that triggered it)
