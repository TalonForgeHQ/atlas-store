2. The 5 audit gaps in the public Datadog trust material

2.1 Gap 1 — Davis AI co-pilot + Bits AI Security Analyst end-to-end provenance join-table

Every Davis AI co-pilot decision (Datadog-Q-A + Datadog-Watchdog-alert-triage + Datadog-Notebook-generation + Datadog-Dashboard-generation + Datadog-monitor-recommendation + Datadog-SLO-recommendation + Datadog-Anomaly-Detection-tuning + Datadog-Code-Analysis-recommendation) + every Bits AI Security Analyst detection (Bits-AI-threat-detection + Bits-AI-IOC-analysis + Bits-AI-misconfiguration-detection + Bits-AI-compliance-mapping) needs a 14-column per-decision join-table: datadog_decision_id, davis_ai_session_id, bits_ai_security_analyst_session_id, llm_reasoning_plan_id, llm_tool_call_chain, davis_ai_decision_reasoning_trace, bits_ai_detection_reasoning_trace, downstream_state_change, alert_generated, downstream_pagerduty_jira_servicenow_state, downstream_slack_notification, downstream_github_pr_created, audit_log_entry_hash, worm_retention_until. This join-table is the SOC 2 CC7.2 + EU AI Act Art. 12 (logging for high-risk AI systems) + ISO 42001 §9.4 (AI system logging) + 7-year WORM + quarterly reconstruction drill audit-evidence surface.

2.2 Gap 2 — Davis AI co-pilot training corpus + fine-tune license provenance

Datadog's Davis AI is built on a chain of base models — likely an OpenAI + Anthropic + Meta Llama + Mistral mixture, fine-tuned on Datadog's telemetry corpus (metrics, traces, logs, user-Q-A-pairs, alert-triage-decisions, notebook-generations, dashboard-generations, monitor-recommendations, SLO-recommendations, anomaly-detection-tuning-suggestions, Code-Analysis-recommendations) and RLHF-tuned on Datadog SRE + SecOps feedback. The training-data + fine-tune-data + RLHF-data + constitutional-AI-data + safety-fine-tune-data + tool-use-fine-tune-data license-provenance chain needs to be enumerated per EU AI Act Art. 53(d) (transparency obligations for GPAI models, Aug 2026 enforcement) + ISO 42001 §6.1.4 (data quality for AI systems) + the upstream-base-model-provider's own terms (OpenAI's data usage policy + Anthropic's acceptable use policy + Meta Llama community license + Mistral research license + the third-party telemetry-corpus license used for fine-tuning). Auditors will ask Datadog: "Show me the bill-of-materials for the Davis AI base model + fine-tune corpus + RLHF corpus + safety-fine-tune corpus."

2.3 Gap 3 — Prompt injection + jailbreak detection via Datadog LLM-Observability span attributes

Every Datadog LLM-Observability span (llm.input.messages, llm.output.messages, llm.tool_calls, llm.tool_results, llm.reasoning_plan, llm.span.duration, llm.token_count, llm.model, llm.cost, llm.span.kind, llm.integration.name, llm.workflow.name, llm.prompt.template, llm.response.id, llm.cache_hit) is a potential prompt-injection vector if the span payload contains attacker-controlled text (e.g., a malicious log line ingested into Datadog Logs, then forwarded to Davis AI as part of an alert-triage prompt, then acted on by the LLM as if it were an operator instruction). The prompt-injection detection log needs 5 columns per span per OWASP LLM Top 10 LLM01 (prompt injection) + LLM06 (excessive agency) + NIST AI RMF MEASURE: datadog_span_id, injection_pattern_detected, jailbreak_pattern_detected, system_prompt_extraction_detected, llm_response_quarantine_flag.

2.4 Gap 4 — Cross-tenant Datadog SaaS isolation-evidence

28,000+ customers share the same Datadog SaaS multi-tenant control plane. The isolation-evidence packet (per SOC 2 CC6.1 logical access controls + GDPR Art. 28 processor obligations + HIPAA §164.504(e) business-associate-agreement + FedRAMP Moderate ATO) needs: per-tenant CMK/BYOK optionality (customer-managed-keys via AWS KMS + GCP KMS + Azure Key Vault + HashiCorp Vault integration) + per-tenant data-residency-region-pinning (US-East-1 + US-West-2 + EU-West-1 + EU-Central-1 + AP-Southeast-2 + AP-Northeast-1) + per-completion-no-leakage-evidence (annual third-party-attested penetration test + SOC 2 CC6.1 audit evidence + GDPR Art. 32 technical-organizational-measures evidence + HIPAA §164.312(a)(2)(iv) encryption-and-decryption audit log). Auditors will ask: "Show me the proof that Amazon's Datadog tenant cannot see Stripe's Datadog tenant's Davis AI Q-A history."

2.5 Gap 5 — Datadog-LLM-Observability-to-downstream-OpenAI/Anthropic/AWS-Bedrock/Google-Vertex-AI billing join-table

Every Datadog LLM-Observability span needs a join-table to the downstream LLM-platform's billing record (per SOC 2 CC7.2 + EU AI Act Art. 12 + SEC 17a-4 WORM retention for financial-relevant AI decisions + EU AI Act Annex III §4 high-risk classification per Art. 6 + 14 + 27 + 43 + 72 + Aug 2026 GPAI enforcement).
