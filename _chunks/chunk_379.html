12-column end-to-end provenance reconstruction

Per AI-feature + per board session + per AI-write + per human-approval record, link these 12 evidence rows:

  1. per-board-session-id (Lucidspark board identifier, immutable)
  2. per-participant-id (SSO identity assertion from SAML 2.0 IdP: Okta / Entra ID / Google Workspace / Ping / OneLogin / etc.)
  3. per-AI-feature (Collaborative AI / Auto-Summarize / AI Cluster / AI Mind-Map / AI Action Items / AI Diagramming / AI Facilitation Prompts / AI Templates)
  4. per-prompt-id (the literal text + timestamp + session + participant that triggered the AI feature)
  5. per-model-id (the Lucid AI model + version + temperature + system-prompt + retrieval-augmented-context)
  6. per-Lucid-AI-plan (the structured plan the model emitted + tool-call sequence + parameter map)
  7. per-tool-call-id (the Lucidspark-internal API call the plan dispatched, with full input/output payload)
  8. per-mutation-id (the sticky / frame / text / shape / template / cluster / summary / mind-map / diagram / action-item / board-frame / board-page record that was created, updated, or deleted, immutable version-history entry)
  9. per-human-approval-id (the immutable evidence row recording that a human participant accepted / rejected / edited the AI-write before it committed to the board)
  10. per-audit-log-entry (Lucid audit-log row, retrievable via Lucid administrative API, retained per the SOC 2 + ISO 27001 + ISO 27701 + FedRAMP retention schedule)
  11. per-prompt-injection-detection-signal-id (when an untrusted input is flagged by the Lucid AI safety classifier, the signal row captures the input span + classifier score + reviewer routing)
  12. per-Enterprise-tenant-id (the workspace / team / Enterprise-tenant boundary that isolates the AI-write from other workspaces per SOC 2 CC6.1 + ISO 27001 A.13.2.1 + ISO 27701 + GDPR Art. 28 + Schrems II SCC + FedRAMP Moderate AC-4 Information flow enforcement)
