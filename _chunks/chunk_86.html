3. Gap #1 — vector write to downstream LLM provenance

SELECT
  v.vector_id,
  v.collection_id,
  v.operation,
  v.hnsw_graph_mutation_id,
  v.quantization_job_id,
  v.payload_filter_decision_id,
  q.hybrid_search_query_id,
  q.dense_score, q.sparse_score, q.bm25_score,
  e.embedding_call_id,
  l.llm_call_id,
  d.downstream_state_change_id
FROM qdrant.vector_audit_log v
LEFT JOIN qdrant.query_audit_log q ON v.vector_id = q.vector_id
LEFT JOIN downstream.embedding_call e ON q.embedding_call_id = e.call_id
LEFT JOIN downstream.llm_call l ON q.llm_call_id = l.call_id
LEFT JOIN downstream.state_change d ON l.call_id = d.llm_call_id
WHERE v.timestamp BETWEEN :start AND :end;

This is the table a buyer asks for when Qdrant-backed RAG output changes a customer record, support ticket, credit decision, clinical summary or regulated workflow.

4. Gap #2 — hybrid-search score explainability

Hybrid search blends dense-vector similarity, sparse-vector terms, BM25/payload text signals and filters. Audit evidence needs score components and ranking deltas so a reviewer can reconstruct why a specific chunk was retrieved.

5. Gap #3 — OSS/proprietary provenance split

Qdrant OSS is Apache-2.0; Qdrant Cloud, Hybrid Cloud, Enterprise, Cloud Inference and Edge carry vendor control-plane surfaces. The audit artifact needs to separate source-available evidence, proprietary evidence and customer-operated evidence.

6. Gap #4 — tenant isolation and payload-filter evidence

For Cloud and Hybrid Cloud, buyers need per-tenant evidence: collection ACLs, API key scope, payload-filter enforcement, snapshot access, backup export, replication topology and VPC/private-link boundaries.

7. Gap #5 — WORM retention and cost attribution

Vector writes, deletes, rebuilds, quantization jobs, Cloud Inference calls, snapshot exports and downstream LLM calls need a WORM-retained cost and evidence table for SOC 2 CC7.2 + EU AI Act Art. 12 + SEC 17a-4 + IRS §6001.

8. Buyer checklist for Qdrant audit-prep

  1. Can Qdrant export vector mutation logs to customer WORM storage?
  2. Can hybrid-search score components be reconstructed per query?
  3. Can HNSW graph mutation and quantization jobs be linked to vector IDs?
  4. Can payload-filter allow/deny decisions be replayed?
  5. Can tenant isolation evidence be exported for Qdrant Cloud and Hybrid Cloud?
  6. Can snapshots, backups and replication events be tied to collection IDs?
  7. Can OSS Apache-2.0 components be separated from proprietary Cloud controls?
  8. Can downstream embedding and LLM calls be joined to retrieval IDs?
  9. Can prompt-injection and retrieval-poisoning attempts be logged per payload?
  10. Can cost attribution be joined to vector write/search/inference events?
  11. Can human review override decisions be retained for Annex III workflows?
  12. Can GDPR processor records map to collection and tenant boundaries?
  13. Can customer-operated Kubernetes/on-prem deployments preserve the same evidence schema?
  14. Can the evidence export pass a quarterly reconstruction drill?
