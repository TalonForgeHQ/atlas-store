Applied example — Stack AI sub-agent handoff provenance

Stack AI's drag-and-drop composer is a multi-agent system: a single user prompt can trigger Agent A (intent classifier) → Agent B (data retriever) → Agent C (response composer) → Agent D (human-in-the-loop approver). Under EU AI Act Art. 14, the audit trail for a multi-agent chain must capture:

  • Agent invocation — which agent invoked which + timestamp + which user's workspace + which prompt slot
  • Context handoff — what intermediate state was passed between agents (e.g. intent label + retrieved-doc-id + response draft)
  • Human-oversight event — which approver + which agent step + which accept/edit/reject decision + content-hash
  • Sub-processor invocation per agent — which LLM + which region + which retention window per agent call

The binder documents this for Stack AI specifically, with concrete examples for the 3 most common agent chains: customer-support ticket triage, sales-outreach + CRM-update, and operations + workflow automation.
