3. The 12-Column Per-Call Provenance Join-Table (the core audit-trail artifact)

The single artifact every Fortune-500 buyer + EU regulator + state AG + HIPAA auditor will ask Bland for is the per-call provenance join-table that links the 12 audit signals below. Without this join-table, the evidence lives in 6-8 separate systems (call-recording store + agent-decision log + Salesforce/HubSpot write API + EHR write API + payment-process API + consent-management platform + disclosure-management platform + redaction-pipeline) and reconstructing a single call takes 2-4 hours of manual correlation. With this join-table, the auditor can SELECT * FROM per_call_audit WHERE call_id = 'X' and get a single row in 200ms.

  1. per-call-ANI-or-WebRTC-session-id — the calling-party identity (or web-RTC session for browser-based calls); the canonical join-key for all downstream tables.
  2. per-call-live-transcript — timestamped + speaker-turn-segmented live transcript; full text + per-word timing + per-turn audio-snip-hash; stored in WORM for 6yr (HIPAA) / 5yr (GLBA) / 4yr (FDCPA).
  3. per-call-agent-decision-log — per-turn: intent-classification-decision + tool-call-name + tool-call-arguments + retrieved-doc-snippet-hash + retrieved-doc-source + agent-response-text + agent-response-hash; linked to Bland's underlying LLM-call + retrieval-call + tool-call IDs.
  4. per-call-source-system-write — any write to Salesforce-lead / Salesforce-opportunity / HubSpot-contact / HubSpot-deal / calendar-booking (Google Calendar / Outlook) / EHR-note-append (Epic / Cerner / Athena) / payment-process (Stripe / Adyen / Braintree); includes pre-write-state + post-write-state + write-API-call-id + write-API-response-code + write-API-timestamp.
  5. per-call-AI-disclosure-timestamp — the timestamp at which the AI-disclosure script began speaking; the spoken-text-transcript (the actual words spoken) + the audio-fingerprint (a hash of the audio waveform for the first 10-15s of the disclosure) to prove the disclosure actually played, not just existed in the prompt.
  6. per-call-human-handoff-evidence — if escalation triggered: handoff-trigger-reason + handoff-trigger-decision-id + handoff-target-human-agent-id + handoff-context-bundle (the full transcript + decision log up to the handoff point) + handoff-timestamp.
  7. per-call-TCPA-prior-express-consent-link — the consent-record-id from the consent-management platform that proves the recipient gave prior-express-consent for AI-initiated calls; the consent-record-timestamp + the consent-record-scope (specific Bland campaign / generic AI-call consent / revoked-and-when).
  8. per-call-wire-fraud-voice-cloning-detection-signal — real-time voice-print-mismatch-score vs. claimed-account-holder-enrolled-voice-print (if any) + voice-synthesis-detection-score (probability the caller's speech is synthesized) + anomaly-flags (e.g. "voice changed because of a cold" social-engineering pattern) + high-risk-call-flag + downgrade-to-human-handoff-decision.
  9. per-call-voice-biometric-verification-decision — speaker-identity-verification-decision (verified / not-verified / declined-to-verify) + the verification-method (voice-print-match / knowledge-based-auth / out-of-band-OTP) + verification-evidence (the audio-snippet + the matched-enrolled-voice-print-hash).
  10. per-call-PHI-GLBA-FDCPA-field-level-redaction — the per-field redaction decisions: which transcript-fields were redacted (name / SSN-last-4 / DOB / account-number / diagnosis-code / medication-name / etc.) + the redaction-method (regex / NER / dictionary) + the redaction-timestamp + the unredacted-version-stored-separately-in-WORM-for-regulator.
  11. per-call-call-audio-transcript-residency-decision — the data-residency-decision: US-resident / EU-resident / customer-configured-region + the storage-region for both the call-audio + the transcript + the Bland-side-evidence (Bland's audit-log must honor the customer's residency requirement per Schrems II + GDPR Art. 28).
  12. per-call-downstream-state-change — the final state change that resulted from the call: which system + which entity + which fields + which new-values + which-old-values + which-API-call-id + which-API-timestamp. The canonical "what happened as a result of this call" record.

4. 5 Audit Gaps Bland's Public Material Doesn't Address

  1. Per-call join-table is the auditor's first ask — the 12-column join-table above. SOC 2 CC7.2 + HIPAA §164.312(b) + EU AI Act Art. 12 + California SB 1001 §3(a)(2) + GLBA §501(b) + FDCPA §803(c) all require this evidence at per-call granularity. The naive audit-trail that captures call-recordings + agent-decision-logs in separate systems fails the auditor's first ask.
  2. Wire-fraud + voice-cloning defense at the voice-input layer — the IVANS / Pindrop 2024 research showed voice cloning can defeat voice-biometric verification in under 6 seconds. Bland must (a) detect speaker-identity anomaly in real-time, (b) flag call as high-risk + downgrade to human-handoff, (c) refuse to execute any write action (payment, account change, PHI disclosure) until human-verified second factor is collected. OWASP LLM Top 10 #1 (Prompt Injection) + EU AI Act Art. 15 (accuracy/robustness/cybersecurity) + FFIEC auth-guidance for voice channels all require this.
  3. State-by-state AI-disclosure compliance — 12 US states with AI-disclosure statutes (CA SB 1001, CO SB 24-205, IL AI Video Interview Act, TX, NY + 7 others). Disclosure scripts differ by state: some require "this is an AI" within first 10 seconds, some before any substantive statement, some before PII capture. The audit gap is jurisdiction-detection + script-version-control + per-call-compliance-log.
  4. PHI / GLBA / FDCPA field-level redaction in transcript store — when a call captures PHI (HIPAA) or financial-account info (GLBA) or debtor-communication content (FDCPA), the transcript-store's PII/PHI/financial-info must be redacted at field-level for downstream use (analytics, model-training, agent-eval) while keeping the unredacted version in the audit-log WORM store for the regulator-required retention window. SOC 2 CC6.7 + HIPAA §164.514(b) min-necessary + GLBA §501(b) + FDCPA §803(c) retention all require this.
  5. Multi-region call-audio + transcript residency — when a regulated EU customer asks Bland for the call-audio + transcript export (GDPR + EU AI Act Art. 12), the export must honor their EU-only residency requirement, with Bland-side evidence also EU-resident (not just the customer's data). GDPR Art. 28 (processor obligations) + Schrems II + EU AI Act Art. 10 (data governance) all require this.
