The 18-column Trino-Query-Span provenance join-table

SOC 2 CC7.2 + EU AI Act Art. 12 + ISO 42001 9.4 require an end-to-end join-table per Trino query that touches an AI-pipeline. Reference schema:

CREATE TABLE trino_query_provenance (
  query_id                  UUID PRIMARY KEY,
  tenant_id                 UUID NOT NULL,
  user_id                   UUID NOT NULL,
  client_ip                 INET,
  client_user_agent         TEXT,
  query_state               TEXT,  -- QUEUED, PLANNING, STARTING, RUNNING, FINISHED, FAILED
  query_text                TEXT,
  query_type                TEXT,  -- SELECT, INSERT, EXPLAIN, etc.
  session_properties        JSONB, -- Trino session flags (query_max_run_time, etc.)
  resource_group            TEXT,  -- Trino resource group for quota-tracking
  coordinator_node          TEXT,
  worker_nodes              TEXT[],
  -- 18 columns below are AI-pipeline-specific:
  catalog_name              TEXT,  -- hive, iceberg, polaris, glue, delta, postgres
  schema_name               TEXT,
  table_name                TEXT,
  iceberg_snapshot_id       BIGINT,-- Iceberg snapshot ID this query read
  iceberg_manifest_list     TEXT,  -- S3/GCS/ABFS path to manifest list
  polaris_principal         TEXT,  -- Polaris catalog principal (auth principal)
  polaris_role              TEXT,  -- Polaris role-based access decision
  polaris_grant_chain       JSONB, -- chain of Polaris GRANTs from principal to table
  paimon_snapshot_id        BIGINT,-- Paimon snapshot ID (if applicable)
  paimon_bucket             TEXT,  -- Paimon bucket for streaming-data
  trino_event_listener_decisions JSONB, -- per-query access-control decisions from event-listener plugins
  downstream_llm_call_ids   UUID[],-- LLM calls this query fed (RAG pipeline)
  downstream_agent_decision_ids UUID[], -- Agent decisions that consumed this query result
  ai_disclosure_label       TEXT,  -- EU AI Act Art. 50 transparency label if result was fed to AI agent
  wrm_retention_until       TIMESTAMPTZ, -- WORM retention deadline per SEC 17a-4 / IRS 6001
  created_at                TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_trino_query_tenant ON trino_query_provenance(tenant_id, created_at);
CREATE INDEX idx_trino_query_iceberg_snap ON trino_query_provenance(iceberg_snapshot_id);
CREATE INDEX idx_trino_query_polaris_grant ON trino_query_provenance USING GIN (polaris_grant_chain);

The auditor will pick a random AI-pipeline query, gather these 18 columns plus the 12 columns from the Iceberg snapshot below, and verify that the end-to-end reasoning-chain reconstructs in under 30 minutes. Most open-source Trino deployments today fail this test at column 14 (polaris_grant_chain) — they don't capture the chain of GRANTs from the principal to the table, so the auditor can't reconstruct the access-control decision.

The 14-column Iceberg-Snapshot provenance table

For every Iceberg table that holds AI-pipeline data (training-corpus, RAG-embeddings, agent-reasoning-chain-traces, prompt-injection-detection-log, model-card-governance-log, downstream-state-change audit-log), the SOC 2 CC6.7 + EU AI Act Art. 10 + ISO 42001 6.1.4 audit requires a 14-column provenance table:

CREATE TABLE iceberg_snapshot_provenance (
  snapshot_id              BIGINT PRIMARY KEY,
  table_uuid               UUID NOT NULL,
  tenant_id                UUID NOT NULL,
  parent_snapshot_id       BIGINT,
  operation                TEXT,  -- append, overwrite, delete, replace
  manifest_list_uri        TEXT,
  summary                  JSONB,  -- Iceberg summary map (operation, spark.app.id, etc.)
  schema_id                INT,
  partition_spec_id        INT,
  default_spec_id          INT,
  last_column_id           INT,
  -- 5 AI-pipeline-specific columns below:
  ai_workload_kind         TEXT,  -- RAG-corpus, training-corpus, agent-trace, prompt-injection-log, model-card-log
  ai_training_corpus_license TEXT, -- EU AI Act Art. 53(d) training-data license provenance
  ai_rag_embedding_model   TEXT,  -- the model that produced the embeddings stored in this snapshot
  ai_prompt_injection_scan_id UUID, -- linkage to prompt-injection-scan results
  wrm_retention_until      TIMESTAMPTZ
);

The auditor will ask: "Show me the Iceberg snapshots that hold training-corpus rows; for each snapshot, show me the Art. 53(d) license provenance + the embedding-model used + the prompt-injection-scan results." Most open-source Iceberg deployments fail at column 11 (ai_workload_kind) — they don't tag which snapshots hold AI-training data vs user-data, so the auditor can't isolate Art. 53(d) scope.

5 audit-gap categories every open-source SQL engine must close

Gap 1 — Trino session-property + resource-group provenance

Every Trino query that touches AI-pipeline data must capture session_properties (the 200+ Trino session flags that control query behavior: query_max_run_time, query_max_stage_count, redact_aware_session_properties, redact_aware_partitions) and resource_group (the Trino resource group the query was assigned to). The auditor will ask: "What was the resource-group quota for this AI-pipeline query? Did the query exceed the quota? If so, was the quota exceeded-quota-event logged?" The fix: enable trino.event-listener plugin that captures these on every query.

Gap 2 — Apache Polaris grant-chain reconstruction

Apache Polaris (incubating) is the new Iceberg REST catalog from Snowflake + AWS + Google + Dremio + Tabular + Netflix + Apple + LinkedIn. The catalog enforces RBAC via GRANT chains (Principal → Role → Table-Privilege). For SOC 2 CC6.1 + GDPR Art. 28 + HIPAA + EU AI Act Art. 10 audit, the deployment must capture the entire grant-chain at query-time, not just the final GRANT. The fix: use polaris-admin-tool --export-grants at query-time and store the export in the audit log.

Gap 3 — Apache Iceberg snapshot isolation-evidence

For multi-tenant Iceberg deployments, the auditor will ask: "Show me the per-tenant snapshot-isolation evidence for the last 30 days." The Iceberg REST catalog (Polaris / Nessie / Gravitino) must enforce per-tenant snapshot-isolation, and the audit log must capture per-tenant snapshot ID + per-tenant manifest-list-path + per-tenant partition-spec. The fix: enforce via catalog-credential-vending per-tenant and capture per-query.

Gap 4 — Apache Paimon streaming-lakehouse audit

Apache Paimon (formerly Flink Table Store) is the streaming lakehouse format. For AI-pipelines that consume streaming data (real-time RAG ingestion, real-time agent-decision-trace, real-time prompt-injection-detection-log), the Paimon bucket + Paimon snapshot + Paimon changelog-producers must be captured. The fix: enable paimon.audit-log and capture per-snapshot.

Gap 5 — Cross-engine WORM retention for AI-pipeline data

SOC 2 CC7.2 + EU AI Act Art. 12 + SEC 17a-4 (financial-services) + IRS 6001 (tax-records) + HIPAA §164.316(b)(2)(i) require WORM (write-once-read-many) retention for AI-pipeline audit-log data. Most open-source Trino / Iceberg / Polaris deployments store audit logs in mutable Postgres / MySQL — which fails WORM. The fix: store audit logs in AWS S3 Object Lock (Compliance mode) + Azure Blob Immutable Storage (Legal Hold) + GCS Bucket Lock (Retained) with per-record retention-until timestamp. The wrm_retention_until column in the join-tables above is the bridge to the WORM store.
