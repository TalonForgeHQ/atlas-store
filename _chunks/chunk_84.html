Audit gap #1 — End-to-end Sierra conversation-turn + tool-call + RAG-retrieval + guardrail-trigger + agent-evaluation + human-handoff + downstream Zendesk / Intercom / Salesforce provenance join-table

The auditor wants a single 16-column provenance join-table that links sierra_conversation_turn.conversation_id to sierra_tool_call_decision.tool_call_id to sierra_rag_retrieval_decision.retrieval_id to sierra_agent_guardrail_trigger.guardrail_id to sierra_agent_evaluation_run.eval_run_id to sierra_human_handoff_decision.handoff_id to sierra_voice_call_transcript.call_id to sierra_chat_message.message_id to sierra_email_thread.thread_id to downstream_zendesk_ticket.ticket_id to downstream_intercom_conversation.conversation_id to downstream_salesforce_case.case_id to downstream_snowflake_state_change.event_id to downstream_databricks_state_change.event_id to downstream_bigquery_state_change.event_id to downstream_redshift_state_change.event_id for any agent-driven action that resulted in a downstream state change. The SOC 2 CC7.2 evidence + EU AI Act Article 12 (logging) requirement is that every agent-driven decision be reproducible from immutable evidence — Sierra Agent Studio has the lineage at runtime, but the export to a SOC 2 evidence format (CSV+JSON to WORM) is a customer-side script, not a one-click Sierra feature. This is the same audit-evidence gap that surfaces at Traceloop 179 (per-OTel-span) + AgentOps 153 (per-LLM-call-chain) + Helicone 155 (per-LLM-call-chain) — the join-table requirement is universal across the ai_agents_infra vertical, but lands harder at Sierra because Sierra is the customer-facing surface (the agent decides the refund, not the developer).

SELECT
  sct.conversation_id, sct.turn_id, sct.timestamp_utc,
  stcd.tool_call_id, stcd.tool_name, stcd.tool_args_hash,
  srrd.retrieval_id, srrd.doc_id, srrd.similarity_score,
  sagt.guardrail_id, sagt.guardrail_type, sagt.action,
  saer.eval_run_id, saer.eval_score, saer.pass_fail,
  shhd.handoff_id, shhd.handoff_reason,
  dz.ticket_id, di.conversation_id AS intercom_conv, ds.case_id,
  dsn.event_id AS snowflake_event, ddb.event_id AS databricks_event
FROM sierra_conversation_turn sct
LEFT JOIN sierra_tool_call_decision stcd USING (turn_id)
LEFT JOIN sierra_rag_retrieval_decision srrd USING (turn_id)
LEFT JOIN sierra_agent_guardrail_trigger sagt USING (turn_id)
LEFT JOIN sierra_agent_evaluation_run saer USING (turn_id)
LEFT JOIN sierra_human_handoff_decision shhd USING (turn_id)
LEFT JOIN downstream_zendesk_ticket dz USING (turn_id)
LEFT JOIN downstream_intercom_conversation di USING (turn_id)
LEFT JOIN downstream_salesforce_case ds USING (turn_id)
LEFT JOIN downstream_snowflake_state_change dsn USING (turn_id)
LEFT JOIN downstream_databricks_state_change ddb USING (turn_id)
WHERE sct.timestamp_utc BETWEEN '2026-03-01' AND '2026-03-31'
ORDER BY sct.timestamp_utc;

Audit gap #3 — Prompt-injection + jailbreak detection (Sierra conversation-turn + tool-call + RAG-retrieval + guardrail-trigger attributes)

OWASP LLM Top 10 LLM01 (prompt-injection) + LLM06 (sensitive-information-disclosure) + NIST AI RMF MEASURE + EU AI Act Article 9 (risk-management) + Article 14 (human-oversight) require that the agent platform detect + log + escalate prompt-injection attempts. Sierra's agent guardrails are good for the customer, but the auditor wants the per-trigger attributes: prompt-injection-pattern-match + jailbreak-pattern-match + sensitive-info-disclosure-attempt + which-tool-call-was-blocked + which-RAG-doc-was-blocked-from-retrieval. Audit gap #4 — Sierra Cloud cross-tenant isolation + Cloud BYOK + Cloud KMS

SOC 2 CC6.1 + GDPR Article 28 + HIPAA + EU AI Act Article 10 require that every customer-experience AI agent platform demonstrate physical or logical tenant isolation with cryptographic evidence. Sierra's multi-tenancy is logical (namespace per customer on shared compute + shared object store), not physical (separate compute + separate object store per tenant). For EU AI Act Annex III high-risk systems + GDPR Schrems II cross-border transfers + HIPAA business-associate requirements, some buyers need per-tenant EU-only residency — i.e. Sierra EU on Sierra EU storage, not Sierra US with EU-data-at-rest encryption. The auditor will ask for the network path diagram + the encryption-at-rest certificate + the key-custody chain + the per-tenant KMS key-evidence + the per-tenant BYOK-evidence. This is the same cross-tenant isolation-evidence gap that surfaces at Traceloop 179 + AgentOps 153 + Helicone 155 + CrewAI 156 — but lands harder at Sierra because Sierra agent conversations carry customer-PII (payment-info, account-info, return-reasons, subscription-state).
