The Sketch AI surface (verified live 2026-07-19)

The Sketch MCP server exposes six core capabilities that an external AI agent can invoke:

  1. Theme-swap — switch light-mode components for dark-mode counterparts without manual swap.
  2. Fill-placeholders — replace placeholder text across every layer with context-aware content (no more lorem ipsum).
  3. Tidy-documents — sort, rename, and align layers automatically; wrangle icon sets; keep Libraries clean.
  4. Build-palette — drop in a photo, get a full set of named color variables that complement it.
  5. Build-designs-from-code — point an AI client at a repo to recreate text styles, colors, UI components, and entire pages inside Sketch.
  6. Create-code-from-designs — take any frame in Sketch and build it out in code with 1:1 visual fidelity.

Each of these is a tool call. Each tool call produces a mutation. Each mutation is auditable — if Sketch publishes the per-tool-call write log.

The 5 audit gaps unique to the Sketch MCP server posture

Gap 1: Per-AI-action provenance

Every MCP-server invocation — every theme-swap Claude executes, every placeholder Cursor fills, every tidy-document Codex runs, every palette Antigravity extracts, every design-from-code VS Code extension regenerates, every create-code-from-designs MCP tool call returns — needs immutable evidence of which MCP-client + which prompt + which model version + which tool call produced the mutation, which document/layer/symbol/text-style/component/Library/theme/color-variable/canvas/frame/code-output was mutated, and which downstream consumer (Claude Code history, Cursor chat log, VS Code extension telemetry, Codex log, Antigravity message log) received the change.

Gap 2: Workspace/document/symbol-library/Library/data-residency isolation

Across the Sketch workspace hierarchy + Library scope + symbol-library scope + cloud-sync scope + local-only-default scope + EU data residency, the EU regulated buyer needs airtight isolation between (a) personal workspace and team workspace, (b) local-only documents and cloud-synced documents, (c) EU-resident data and US-resident data, (d) sketch.com-managed encryption and user-managed encryption, and (e) MCP-server running and MCP-server off. SOC 2 CC6.1 + GDPR Art. 28 + ISO 27001 + ISO 27701 + Dutch + EU jurisdiction.

Gap 3: Prompt-injection defense for untrusted MCP-client payloads

The Sketch MCP server accepts prompts from Claude Code, Claude Desktop, Cursor, VS Code, Codex, and Antigravity. Each of these MCP-clients is a separate prompt-injection attack surface. If a designer copies a snippet from an untrusted web page into Claude Code, the MCP server executes whatever the model returns. The defense playbook needs to cover prompt injection through every MCP-client channel with formal evidence per OWASP LLM Top 10 LLM01+LLM03+LLM06+LLM08 + MITRE ATLAS + EU AI Act Art. 14 + Art. 50.

Gap 4: Deletion/retention/rollback/version-history/MCP-server-write-evidence

When a designer (or an AI agent acting on the designer's behalf) deletes a layer, fills a placeholder, swaps a theme, or tidies a document, the propagation to the connected surfaces — Sketch version history, cloud-sync history, MCP-server write log, Claude Code conversation history, Cursor chat log, VS Code extension telemetry, Codex log, Antigravity message log — needs to be auditable with retention windows and rollback capability.

Gap 5: Cost attribution per agent/model/tool-call/workspace/document/MCP-client/tenant

Across Sketch MCP + Sketch AI + Claude Code + Claude Desktop + Cursor + VS Code + Codex + Antigravity, the EU regulated buyer needs per-MCP-client cost attribution. Currently, no per-Sketch-workspace invoice trail exists in the audit log that attributes MCP-server cost back to the specific MCP-client that incurred it.
