Gap #2 — Prompt-injection + voice-cloning + tool-call-poisoning defense

Voice-AI agents ingest raw audio (highest-trust input modality in any AI system), pass it through a speech-to-text model, then an LLM, then a TTS model, then output to a phone line — with multiple tool-call hops in the middle (CRM write, calendar book, payment capture, EHR note append). Every hop is a prompt-injection surface per OWASP LLM Top 10 LLM01.

The 6 specific voice-AI injection vectors:

  1. Voice cloning at input layer — the IVANS/Pindrop 2024 research showed voice cloning can defeat voice-biometric verification in under 6 seconds. Defense requires voice-print-mismatch-score vs claimed-account-holder-enrolled-voice-print + voice-synthesis-detection-score + high-risk-call-flag + downgrade-to-human-handoff-decision + refuse-write-action-without-second-factor.
  2. TTS-segment poisoning — the TTS output stream can be poisoned to inject out-of-band signaling or social-engineering content. Defense requires per-segment watermarking.
  3. Tool-call-poisoning via LLM hallucination — agent decides to call crm.write() with wrong account_id. Defense requires per-tool-call-id linkage to per-crm-write-id for downstream rollback.
  4. Prompt-injection via transcript poisoning — caller embeds "ignore prior instructions" in transcript. Defense requires transcript-sanitization layer before LLM context.
  5. Billing-charge-bypass via tool-call-bypass — agent decides to skip the payment step. Defense requires per-downstream-billing-id linkage.
  6. CRM-write-bypass via tool-call-bypass — agent decides to skip the CRM update. Defense requires per-CRM-write-id linkage.

Gap #4 — PHI / GLBA / PCI / FDCPA field-level redaction

Voice-AI calls in healthcare (PHI), financial services (GLBA), payments (PCI), and collections (FDCPA) all carry regulated data. That data must be redacted in the transcript store used for downstream analytics + model-training + agent-eval, but unredacted in the audit-log WORM store for regulator-required retention windows.

Gap #5 — EU data-residency for call-audio + transcripts

EU customers (GDPR + EU AI Act) require call-audio + transcript storage in EU-resident infrastructure. This means both customer-data residency AND per-tenant-isolation in shared EU regions, with Schrems-II-compliant cross-border-transfer SCCs for any US-resident processing.

Required evidence: per-call-EU-region-routing-decision + per-tenant-EU-isolation-flag + per-cross-border-transfer-sccs-version + per-VPC-peering-id + per-tenant-KMS-key-id (CMK/BYOK) + per-WORM-retention-flag in EU-resident WORM store.
