2. The audit-trail join-table pattern (canonical schema)

The shape auditors query under SOC 2 CC7.2 + EU AI Act Art. 12 + ISO 42001 9.4 + GDPR Art. 28 + NIST AI RMF MEASURE is a single join-table row per AI-cell execution. For Hex, the 14 columns are:

  1. notebook_id — per-Hex-notebook-row identifier (UUID v4)
  2. cell_id — per-cell-row within a notebook (UUID v4)
  3. ai_cell_execution_id — per-AI-cell-run (UUID v4)
  4. llm_call_id — per-LLM-call (model + provider + prompt-hash + response-hash + token-count + cost)
  5. mcp_tool_call_id — per-MCP-tool-call (tool-name + args-hash + response-hash + latency)
  6. warehouse_query_id — per-warehouse-query (Snowflake QUERY_ID + BigQuery job_id + Databricks run_id + dbt invocation_id)
  7. warehouse_source_data_asset_id — per-source-table-or-view (Snowflake table_id + BigQuery table_ref + Databricks table_name + dbt model_id)
  8. notebook_uploaded_data_asset_id — per-customer-uploaded-CSV-or-file (object-store key + hash + size + retention-policy)
  9. user_id + session_id + tenant_id — per-execution principal + SSO claim + workspace
  10. ai_bill_of_rights_eval_id — per-AI-Bill-of-Rights-evaluation (NIST AI RMF GOVERN + MAP)
  11. nist_ai_rmf_measure_event_id — per-NIST-AI-RMF-MEASURE-event (ISO 42001 9.4)
  12. owasp_llm_top10_eval_result — LLM01-LLM10 flags per execution
  13. mitre_atlas_eval_result — ATLAS technique flags per execution
  14. eu_ai_act_art12_record_evidence_hash — WORM log hash (sha256) per Art. 12 + Aug 2026 GPAI enforcement
