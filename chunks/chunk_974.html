The 5 evidence gaps

Gap 1 — Replay join-table for human + non-human + AI-agent decisions

An auditor should be able to pull, per identity, per non-human identity, and per AI agent, the application onboarding record, joiner-mover-leaver event, IGA policy decision, SoD evaluation, PAM session or vault entry, ISPM finding, AI-agent action, and tenant-isolation key, and reconstruct a single audit-trail in under an hour.

Gap 2 — Cross-product identity handoff

When a single identity moves from workforce to privileged user to non-human to AI agent, the handoff should preserve the policy version, approver chain, and tenant boundary. Cross-product identity state should not be reconstructed by an auditor from separate per-product logs.

Gap 3 — ISPM for AI Agents action audit-trail

Each AI-agent decision — prompt, model, retrieval, tool invocation, and human override — should produce a signed action record. The "Identity Security for AI" framing on the home page is broad; an auditor should map it to a specific action-record schema with retention and tamper-evidence.

Gap 4 — External Identity Management boundary

External identities (suppliers, contractors, partners) should have a discrete lifecycle, a discrete SoD evaluation, and a discrete access-review cadence. A federated B2B identity should not be audit-equivalent to a workforce identity.

Gap 5 — Evidence export under SOC 2 / ISO 27001 / ISO 42001

An auditor should be able to export, for any audit period, a coherent evidence bundle mapped to SOC 2 Common Criteria, ISO/IEC 27001 Annex A controls, and ISO/IEC 42001:2023 AIMS clauses. The bundle should not require manual reconciliation across IGA, PAM, ISPM, and External Identity product lines.
