Why this ledger matters for AI Agent builders

Most AI agent builders reach for an eval framework because the framework gives them a passing score on a small held-out set. The procurement question is different: when the agent ships, can you re-run the same scoring rubric against the production-traffic trace, with a regression baseline pinned, a Clickhouse table query that joins trace + observation + score + cost, and an MCP-driven SKILL.md / CLI access pattern for coding agents? That's a different question than "did the eval pass in dev." Langfuse's Tracing + Evaluation + Prompt Management + Datasets + Experimentation + LLM-as-a-Judge primitives are designed to answer it — but only if the receipt design captures the join between trace identity, prompt version, score reasoning, and the self-host deployment fingerprint.
