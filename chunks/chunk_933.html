2. The Cisco ownership lens — what changed in March 2024

Verbatim first-party /about-splunk.html 2026-07-22: "Supercharged by Cisco, Splunk extends resilience across the entire tech stack, spanning networks, endpoints, cloud environments, and applications". Cisco closed the $28B acquisition of Splunk in March 2024. For procurement, that changes the diligence lane in five concrete ways:

  • Cisco parent compliance posture applies alongside Splunk's own SOC 2 / ISO 27001 / ISO 27017 / ISO 27018 stack — including Cisco's FedRAMP Moderate + DoD IL4 authorizations, which Splunk customers now inherit via the parent's ATO letter.
  • AppDynamics cross-product credential scoping — AppDynamics was a Cisco product (acquired 2017) before Splunk joined Cisco. The Splunk Observability nav now lists AppDynamics verbatim ("Optimize apps with full-stack insight") — so a customer touching both AppDynamics + Observability Cloud inside the same Cisco tenant needs to verify per-product credential scope doesn't bleed.
  • Splunk MCP Server inherits Cisco's Network Edge + Hypershield threat surface — every MCP tool-call that traverses Splunk's index is now potentially visible to Cisco's security-product telemetry, which has its own Reg FD + DORA + NIS2 disclosure implications.
  • Cisco ThousandEyes + Splunk APM integration — ThousandEyes was Cisco's network-observability SKU pre-acquisition. Procurement should ask whether ThousandEyes internet-monitoring data is now visible inside Splunk APM, and what the per-tenant isolation looks like there.
  • Cisco Talos threat-intel feed — Talos is Cisco's threat-intelligence unit. With Splunk now under Cisco, the question is whether Talos threat-feed data is fed back into the customer's Splunk index by default (it is — see Cisco Talos + Splunk ES integration docs), and whether customers can opt out without losing other features.
