The 5 audit gaps every LiveKit Agents SDK deployment must close (2026)

Gap #1 — End-to-end per-room provenance (60-column join-table)

Auditors will pull the canonical join-table that links every per-room-id to every downstream artifact: per-participant-id → per-track-id → per-audio-frame-id → per-VAD-segment-id → per-LLM-call-id → per-prompt-version-id → per-completion-id → per-TTS-segment-id → per-tool-call-id → per-handoff-event-id → per-recording-id → per-egress-job-id → per-CRM-write-id → per-downstream-billing-id. Most LiveKit deployments stitch this from 4-7 different services (LiveKit Cloud room events + LiveKit Agents SDK LLM-call logs + LiveKit Egress S3 metadata + downstream CRM webhook logs) and the joins break under audit pressure.

Fix: ship a single per-room provenance view in your audit-log export that joins LiveKit Cloud room-events + per-room-id + LiveKit Agents SDK session-events + per-LLM-call-id + LiveKit Egress S3 metadata + per-egress-job-id + downstream CRM webhook-logs + per-CRM-write-id in one queryable surface. Map to SOC 2 CC7.2 (system operations monitoring) + HIPAA §164.312(b) (audit controls) + EU AI Act Art. 12 (record-keeping) + GLBA §501(b) (records retention).

Gap #2 — Prompt-injection + VAD-segment-poisoning + tool-call-poisoning + WebRTC-SFU-bypass defense

LiveKit's VAD-segment + audio-frame pipeline is a unique injection surface: an attacker can inject malicious audio-frame-id content (voice-cloned speech, hidden TTS segments, synthetic VAD-burst attacks) that flows through VAD → STT → LLM-call → tool-call without the per-LLM-call-id provenance record reflecting the poisoning. Combined with WebRTC-SFU-bypass (where a malicious participant-id can impersonate a different participant-id within a room), this is a real attack surface.

Fix: implement per-audio-frame-id + per-VAD-segment-id + per-LLM-call-message-id poisoning detection signals (e.g. TTS-detection model + voice-cloning-detection model + per-participant-id MFA-challenge). Map to OWASP LLM Top 10 LLM01 (prompt injection) + LLM03 (training data poisoning) + LLM04 (model DoS) + LLM06 (sensitive information disclosure) + LLM08 (excessive agency) + EU AI Act Art. 15 (accuracy, robustness, cybersecurity) + MITRE ATLAS AML.T0051 (LLM Prompt Injection) + AML.T0054 (LLM Output Manipulation).

Gap #3 — Multi-region SFU data-residency (Schrems II + GDPR Art. 28 + EU AI Act Art. 10)

LiveKit Cloud defaults to multi-region SFU routing (US + EU + APAC). For EU customers, the per-room-id audio-frame-id + recording-id + egress-job-id data may transit or be stored in non-EU regions — violating Schrems II + GDPR Art. 28 (processor obligations) + EU AI Act Art. 10 (data governance). The per-room-id region-routing decision is what auditors will pull.

Fix: ship a per-room-id region-routing log that records the SFU region + recording-storage region + egress-job region + downstream-CRM-write region for every room. Provide customer-controllable region pinning (EU-only for EU customers). Map to Schrems II + GDPR Art. 28 + EU AI Act Art. 10 + ISO 27018 + ISO 27701.

Gap #4 — PHI / HIPAA-eligible Telephony + SIP + Egress pipeline

LiveKit Telephony + LiveKit SIP + LiveKit Egress handle the per-room-id → per-participant-id → per-track-id → per-audio-frame-id → per-recording-id → per-egress-job-id chain that often carries PHI for telehealth + HIPAA-regulated voice-AI agents. Without BAA-ready surface + per-room-id PHI-flag + per-recording-id encryption + per-egress-job-id PHI-tag + per-CRM-write-id PHI-redaction, the deployment is not HIPAA-eligible.

Fix: ship per-room-id PHI-flag + per-recording-id CMK/BYOK encryption + per-egress-job-id PHI-tag + per-CRM-write-id PHI-redaction hook + BAA-ready LiveKit Cloud configuration. Map to HIPAA §164.312(a)(2)(iv) (encryption) + §164.312(b) (audit controls) + §164.312(e)(1) (transmission security) + §164.514(b) (minimum necessary).

Gap #5 — State-by-state AI-disclosure + TCPA + wire-fraud-voice-cloning detection

12 US states have active AI-disclosure rules (CA SB 1001, CO SB 24-205, IL AIVIA, TX, NY, + 7 others). LiveKit's voice-AI agent stack must ship per-room-id jurisdiction-detection + per-LLM-call-id disclosure-script-version + per-call-id human-handoff-evidence. Plus TCPA prior-express-consent for outbound voice (telemarketing, debt collection). Plus wire-fraud-voice-cloning detection per FTC + FBI guidance (voice-cloning-detection-signal-id).

Fix: ship per-room-id jurisdiction-detection + per-LLM-call-id disclosure-script-version + per-call-id human-handoff-evidence + per-call-id TCPA-prior-express-consent-link + per-audio-frame-id voice-cloning-detection-signal. Map to CA SB 1001 §3(a)(2) + Colorado SB 24-205 + Illinois AI Video Interview Act + TCPA §227 + 47 CFR §64.1200 + FTC Voice Cloning Consumer Protection Act.
