The 2026 cohort of AI agents on AWS — Amazon Bedrock agents, Amazon Q Developer, SageMaker inference endpoints, Bedrock AgentCore runtime agents, AWS App Studio apps, and CodeWhisperer Enterprise — all consume secrets via IAM Role assumption against AWS Secrets Manager. No long-lived API tokens, no plaintext .env files, no external vault hops. The agent's MCP server config can assume the IAM role at runtime and pull the secret fresh per-invocation with zero embedded credentials. This is the wedge the SaaS-only vendors cannot replicate: first-party AWS IAM broker + first-party AWS KMS + first-party AWS CloudTrail + first-party AWS Bedrock AgentCore Identity all integrated at the platform layer.

For AI agent operators who already run on AWS — which is most of the Fortune 500 — the only choice that keeps secrets in the same identity plane as the agent's other AWS API calls is AWS Secrets Manager. Bringing in a SaaS secrets vendor (Akeyless, Doppler) creates a second identity plane, a second audit log, and a second compliance review packet. That's the wedge.

  • You're running AI agents on AWS (Amazon Bedrock + SageMaker + Bedrock AgentCore + Amazon Q + AWS App Studio + CodeWhisperer Enterprise)
