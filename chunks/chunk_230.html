8-step vendor-DD playbook for Bardeen

  1. Per-workflow provenance audit: request the canonical join-table covering per-workflow-id → per-trigger-id → per-action-id → per-browser-step-id → per-scraper-id → per-form-fill-id → per-API-call-id → per-CRM-write-id → per-Slack-message-id → per-webhook-delivery-id → per-billing-event-id → per-credential-access-id lineage. Atlas's audit-template uses 60+ columns; Bardeen-specific events like Magic Box AI-decision steps, Autobook scheduled-job runs, and per-browser-session-id ephemeral-session boundaries must be captured.
  2. Prompt-injection + adversarial-page defense audit: Bardeen reads arbitrary third-party web pages and executes browser actions on behalf of users. Test prompt-injection defense against (a) malicious instructions embedded in scraped page content, (b) per-browser-page-poisoning via attacker-controlled pages visited during scraping, (c) per-scraper-payload-poisoning of extracted data, (d) per-form-fill-value-poisoning where attacker-controlled form values propagate downstream, (e) per-CRM-record-poisoning where poisoned data writes to Salesforce / HubSpot / Pipedrive, (f) per-Slack-message-poisoning where poisoned data posts to internal Slack. Benchmark against OWASP LLM Top 10 LLM01+LLM03+LLM04+LLM06+LLM08 + MITRE ATLAS AML.T0051+AML.T0054 + EU AI Act Art. 15 + NIST AI RMF MEASURE.
  3. Cross-region browser-execution-data-residency audit: for EU + India + Brazil + UAE + Australia + Canada + UK + Singapore + Japan + Philippines customers, verify the per-customer-region selection mechanism for (a) browser-execution geographic routing, (b) scraped-data-storage region, (c) workflow-state-storage region, (d) credential-vault region, (e) audit-log region. Audit against Schrems II + GDPR Art. 28 + EU AI Act Art. 10 + India DPDP Act 2023 + Brazil LGPD Art. 33 + UAE PDPL + Canadian PIPEDA + Singapore PDPA + Philippines DPA + Japan APPI.
  4. HIPAA-eligible Bardeen audit: if your healthcare customers use Bardeen for HIPAA-covered workflows (patient-intake-form-fills, provider-CRM-sync, payer-portal-scraping, clinical-trial-recruitment-screening), verify the BAA-ready configuration: (a) per-credential-id PHI-flag handling, (b) per-scraper-id CMK/BYOK encryption, (c) per-browser-session-id BAA-isolation, (d) per-CRM-write-id PHI-tag propagation. Floor: HIPAA §164.312(a)(2)(iv) + §164.312(b) + §164.312(e)(1) + §164.514(b).
  5. Cross-tenant isolation audit: for Bardeen Cloud + Bardeen Enterprise + Bardeen Self-hosted + per-workspace-id + per-team-id + per-user-id + per-API-key-id + per-credential-id + per-browser-session-id isolation, request (a) most recent SOC 2 Type II report covering CC6.1, (b) GDPR DPA Art. 28 evidence, (c) EU AI Act readiness statement, (d) FedRAMP Moderate readiness roadmap. Especially focus on how user-stored credentials (Salesforce OAuth tokens, HubSpot API keys, LinkedIn session cookies, Gmail App Passwords, Notion API tokens) are isolated across tenants — credential-vault encryption-at-rest + per-tenant CMK boundaries.
  6. No-code-workflow + Magic Box AI-decision explainability audit: Magic Box lets users describe a workflow in natural language and Bardeen generates the workflow. Audit: (a) per-Magic-Box-generation-id LLM-decision-log (which model version, what prompt, what tool calls, what schema-output), (b) per-AI-decision-step explainability for SOC 2 + EU AI Act Art. 13 (transparency), (c) per-workflow-version audit-log capturing every Magic Box regeneration, (d) per-AI-confidence-score-id surfacing in the workflow editor.
  7. Credential-vault + secret-management audit: Bardeen stores OAuth tokens + API keys + session cookies for 100+ SaaS connectors. Audit: (a) credential-vault encryption-at-rest (AES-256 + per-tenant CMK), (b) credential-vault access-log retention (SOC 2 CC7.2 floor: 90 days hot + 1 year cold), (c) per-credential-access-id audit-event covering every read/write/use, (d) credential-rotation workflow, (e) per-user-offboarding credential-revocation SLA, (f) per-compromise-detection credential-revocation automation.
  8. Browser-agent autonomous-execution guardrails audit: Bardeen AI Browser Agent can autonomously navigate + click + fill forms + extract data on arbitrary web pages. Audit: (a) per-step human-approval-required configuration for high-risk actions (e.g. sending emails, posting Slack messages, writing to CRM records, making purchases), (b) per-step dry-run-preview before live execution, (c) per-action-confirmation prompt for irreversible actions, (d) per-step audit-log including screenshot capture + DOM-snapshot, (e) per-budget-id cost-limit to prevent runaway agent spend.
