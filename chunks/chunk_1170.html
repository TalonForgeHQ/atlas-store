First-party product surface (verbatim from argoproj.github.io + GitHub API 2026-07-24)

  • Workflow — Kubernetes CustomResourceDefinition (CRD) representing a single workflow execution instance (spec + status + metadata under the argoproj.io/v1alpha1 API group)
  • WorkflowTemplate — Kubernetes CRD representing a reusable, namespaced workflow definition (spec.statusType / spec.entrypoint / spec.templates / spec.arguments)
  • CronWorkflow — Kubernetes CRD representing a scheduled workflow trigger (spec.schedule / spec.concurrencyPolicy / spec.successfulJobsHistoryLimit / spec.failedJobsHistoryLimit)
  • ClusterWorkflowTemplate — Kubernetes CRD representing a cluster-scoped reusable workflow definition (cross-namespace)
  • WorkflowTaskSet — Kubernetes CRD representing a fan-out / status-aggregation substrate for expanded workflows
  • DAG — first-class orchestration type (tasks + dependencies + directed-acyclic-graph)
  • Step — first-class orchestration type (sequential list of tasks)
  • Parameter — first-class input/output type on every template (spec.arguments.parameters + spec.outputs.parameters)
  • Artifact — first-class per-step artifact type (S3 + GCS + OSS + HTTP + Git + raw) with retention semantics
  • Container + Script + Resource — first-class task types (Pod-as-task primitive)
