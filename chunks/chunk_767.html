Five buyer-facing joins

  1. Chat user → question → retrieved source → bot answer: Slack/Teams user, question text, channel scope, retrieved source (Drive + GitHub + Figma + Notion + Confluence + Tettra wiki), source ACL snapshot, bot answer, LLM sub-processor invoked (OpenAI vs. Anthropic vs. self-hosted), retention window, region routing, and tenant.
  2. Expert-verification escalation and write-back: unverified-question event, subject-matter expert identity, verification timestamp, revision history, Strict-vs-Loose verification mode, downstream re-retrieval flag, Tettra wiki category and subpage written-back to, and the next-aswer effect.
  3. Per-source ACL + integration-credential scope: Drive folder + GitHub repo + Figma file + Notion page + Confluence space mapped per Tettra category + per role + per integration OAuth credential / GitHub App installation; rotation/revocation cascade window; stale-credential-serving-answers detection.
  4. Slack + MS Teams bot data-handling disclosure: per-workspace admin-set scope vs. installing-user-can-read-everything scope; private channel read access; DM read access; threaded conversation read access; ephemeral message read access; EU AI Act Art. 50 transparency-labeling (bot answer watermarked, but the prompt-from-DM is not labeled).
  5. Sub-processor cascade DPA + 14-day change-notification SLA: five first-party integration sub-processors (Google + GitHub + Figma + Notion + Confluence), two LLM sub-processors (OpenAI + Anthropic), two delivery sub-processors (Slack + Teams) — totaling nine sub-processors in the per-answer cascade; flow-down DPA + 14-day change-notification SLA per GDPR Art. 28(2).

Common evidence failures

  • Showing a Tettra bot answer in Slack or Teams without the question text, the channel scope, the channel-scoped role, the retrieved source, the LLM sub-processor invoked, and the verification state at answer time.
  • Logging an expert-verification write-back without the verifier identity, the verification timestamp, the revision history, the Strict-vs-Loose mode, and the downstream re-retrieval flag.
  • Recording a per-source ACL mapping without the integration credential backing it (which OAuth credential + which GitHub App installation + which scope), and without the rotation/revocation cascade window.
  • Allowing a Slack or Teams bot to read private channels or DMs without enumerating the scope, the admin-set boundaries, and the ephemeral-content handling, and without mapping the prompt-from-DM to EU AI Act Art. 50 transparency-labeling.
  • Deleting a Tettra article or revoking a bot-installation while omitting connected-source copies, embeddings, search caches, chat answers, expert-verification events, action traces, replicas, or backups.
