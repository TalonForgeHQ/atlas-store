Five buyer-facing joins

  1. Role → Agent → governed data: account, user/session, default role, privilege snapshot, Agent object/version, model/instructions, semantic view or governed object, tenant, and region.
  2. Thread → Run → event: Thread ID, Run ID, planning event, selected tool, reflection, response, citation, token/cost record, latency, failure, retry, and final state.
  3. Tool → authorization → action: Analyst/Search query, isolated-code execution, custom tool, skill, MCP connector, or web search; identity, credential boundary, policy decision, input/output, downstream mutation, and escalation.
  4. Feedback → evaluation → release: rating or free-text comment, evaluation dataset/version, metric, reviewer, instruction/model/tool change, promoted Agent revision, rollback target, and release timestamp.
  5. Export → deletion: Threads, Runs, events, prompts, citations, tool inputs/outputs, code artifacts, traces, feedback, evaluations, caches, logs, replicas, backups, completion time, and exceptions.

Common evidence failures

  • Showing a final answer without the Thread, Run, event sequence, selected model, and exact tool resources.
  • Logging an MCP or custom-tool invocation without the session role, privilege decision, credential boundary, and downstream mutation.
  • Reporting an evaluation score without dataset version, reviewer, promoted Agent revision, and rollback target.
  • Deleting a Thread while omitting Run events, tool outputs, code artifacts, traces, feedback, evaluation copies, caches, replicas, or backups.
