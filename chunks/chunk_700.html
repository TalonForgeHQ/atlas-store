2. The five-block technical-documentation cascade (Article 53 GPAI provider obligations)

Block 1 — Per-model-version-pinning

For every Mistral model release (Mistral Large 3 + Medium 3 + Small 3 + 7B + Mixtral 8x7B + Mixtral 8x22B + Codestral + Pixtral Large + Pixtral 12B + NeMo + Saba + OCR + Voxtral + 13B + 24B + 70B + 123B), produce an audit JSON with: model-id, model-version, model-release-date, model-weights-license (Apache 2.0 / Mistral Research License / Mistral Commercial License), model-SHA256-weights-hash, per-system-card-version, per-model-card-version, per-Mistral-responsible-AI-policy-version, per-training-data-summary-publication-version, per-Mistral-trust-center-publication-version, per-AI-evaluation-benchmark-versions (hallucination + bias + toxicity + jailbreak), per-Mistral-content-moderation-classifier-version, per-Mistral-guardrails-classifier-version.

Block 2 — Per-deployment-region-pinning

For every Mistral La Plateforme inference call, produce an audit JSON with: deployment-region (EU-Frankfurt-Paris / US / APAC), per-API-key-rotation-log, per-fine-tuning-data-retention-window, customer-data-no-train-flag, content-exclusion-list-active, per-tenant-isolation-token, per-Mistral-VPC-pinning, per-Mistral-content-moderation-API-tenant-isolation, per-inference-tenant-isolation, per-Mistral-system-card-publication-flag, per-Mistral-responsible-AI-policy-version-pinning.

Block 3 — Per-fine-tuning-job-version-pinning

For every Mistral fine-tuning job, produce an audit JSON with: fine-tuning-job-id, fine-tuning-job-version, fine-tuning-dataset-version, fine-tuning-data-retention-window, fine-tuning-data-no-train-default-flag, fine-tuning-content-moderation-classifier-version, fine-tuning-guardrails-classifier-version, fine-tuning-AI-evaluation-benchmark-versions (hallucination + bias + toxicity + jailbreak before/after), fine-tuning-region-pinning, fine-tuning-tenant-isolation-token.

Block 4 — Per-customer-deployment-mode-pinning

For every Mistral customer on EU-Host + EU-AI-Act-Aug-2025-GPAI-Code-of-Practice-signatory + EU-AI-Act-Aug-2-2026-GPAI-provider-compliance, produce an audit JSON with: customer-tenant-id, deployment-mode (La-Plateforme-cloud / Mistral-on-prem / Mistral-edge-iOS / Mistral-edge-Android / Voxtral-audio), per-AI-evaluation-benchmark per-model-version, per-hallucination-benchmark per-model-version, per-bias-benchmark per-model-version, per-toxicity-benchmark per-model-version, per-jailbreak-benchmark per-model-version, EU-AI-Act-Art-53-GPAI-Art-53(1)(a)(b)(c)-compliance-flag, EU-AI-Act-Annex-IV-technical-documentation-ready-flag, GDPR-Art-28-sub-processor-flag, GDPR-Art-27-EU-rep-on-file-flag, UK-GDPR-Art-27-UK-rep-on-file-flag, DSA-compliance-flag, AI-Pact-voluntary-commitment-flag.

Block 5 — Per-third-party-AI-system-provider downstream-customer-audit pass-through

For every Mistral Third-Party-AI-System-Provider auditor (i.e. downstream Fortune 500 + AmLaw 200 + EU-Commission-supplier customer of Mistral La Plateforme), produce an audit JSON with: auditor-customer-tenant-id, auditor-customer-deployment-mode (cloud / on-prem / edge), auditor-customer-region (EU-Frankfurt-Paris / US / APAC / LATAM), auditor-customer-procurement-grade-evidence-bundle-version (which release of Mistral's per-model-version-pinning + per-deployment-region-pinning + per-training-data-summary-publication evidence was last refreshed), auditor-customer-EU-AI-Act-Art-53-GPAI-compliance-flag, auditor-customer-Annex-IV-technical-documentation-flag.
