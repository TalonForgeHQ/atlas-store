Q2 — Per-channel-specific evidence trail

Chat interactions, email interactions, voice calls, and SMS threads have different evidentiary requirements:

  • Chat: per-message + per-tool-call + per-knowledge-base-fetch-id + per-citation-source-URL + per-citation-domain + per-citation-trust-tier
  • Email: per-thread-id + per-message-id + per-recipient-id + per-attachment-id + per-attachment-scan-result + per-DLP-redaction-bit
  • Voice: per-call-id + per-utterance-id + per-speaker-id + per-voice-cloning-detection-signal + per-AI-disclosure-timestamp + per-AI-disclosure-spoken-text + per-TCPA-prior-express-consent-evidence + per-call-recording-bit + per-call-recording-consent-bit
  • SMS: per-thread-id + per-message-id + per-recipient-id + per-short-code-id + per-keyword-id + per-opt-in-timestamp + per-opt-in-source-bit + per-DNC-scrub-result

Q3 — Per-deflection-cost economic evidence

The economic value of an AI customer-support agent is the deflection cost — the dollar amount of human-agent-handle-time saved per conversation. To reproduce this on demand, the vendor must surface:

  • per-conversation-id baseline_handle_time_seconds (average human-agent-handle-time for the same intent-class, last 90 days)
  • per-conversation-id ai_handle_time_seconds
  • per-conversation-id human_handoff_required_bit (TRUE = no deflection)
  • per-conversation-id human_handoff_handle_time_seconds (if TRUE)
  • per-conversation-id deflection_cost_USD = (baseline_handle_time_seconds − (ai_handle_time_seconds + human_handoff_handle_time_seconds if TRUE else 0)) × per-second-human-agent-cost-USD

Q4 — Per-LLM-model-version pinning

When the vendor upgrades from claude-opus-4-2026 to claude-opus-4-2026-Q3 mid-quarter, every prior conversation's evidence-pin must continue to reference the model-version-id that produced the original response. If the evidence-pin only references the vendor's current model, the audit trail is non-reproducible for any conversation older than the latest model upgrade.

This is the single most common audit failure for ai_customer_support_platform vendors in 2026. The fix is per-conversation-id → per-LLM-call-id → per-model-version-id → per-prompt-version-id → per-guardrail-version-id pinning at write-time, not at query-time.

Q5 — Per-deployment-region + per-tenant-KMS-key-id + per-pii-redaction-bit

EU AI Act Art. 26 deployer obligations require per-deployment-region pinning. HIPAA + GDPR + CCPA require per-tenant-KMS-key-id pinning so that tenant-A encryption keys never decrypt tenant-B data. PCI-DSS requires per-pii-redaction-bit on every transcript + every recording + every attachment.

If the vendor cannot answer "which region did this conversation run in, which KMS key encrypted the data at rest, and which PII fields were redacted from the transcript" in a single SQL query, the audit-trail is non-compliant.
