The 14-doc AI-inference evidence binder for Replicate

1. Per-Model-Version Provenance Schema

Every one of the 100K+ models on replicate.com/models has a version hash, training-data cutoff, license, and cog container spec. 2. Per-Inference Audit Trail Spec

Each API call to Replicate captures: input + output + model-id + model-version-hash + GPU-type (A100 / H100 / T4 / L40S / A10G) + region (US / EU / UK / APAC) + retention-window + deletion-cascade receipt. The audit trail is the per-inference equivalent of a database query log — it answers "who called what model, when, with what input, and what came back" with provable non-repudiation.

3. Per-cog-Container Isolation Proof

Every Replicate model runs in an isolated cog container (Docker). Concurrent requests do not share memory, network, or filesystem. The isolation proof demonstrates container-level boundaries via Docker run flags, namespace isolation, cgroup resource limits, and per-container filesystem mounts.

4. LLM Sub-Processor Cascade Disclosure

Replicate uses three primary GPU fleet sub-processors: AWS (EC2 P4d/P5 + Inferentia), GCP (A2/G2 + TPU v4/v5e), and CoreWeave (specialty H100/MI300X pools). The cascade disclosure includes each sub-processor's DPA, security certifications, and data-residency footprint. Plus Replicate's own inference gateway and any model-source upstream (Hugging Face, GitHub).

5. Per-Region Data-Residency Matrix

Customers select US, EU, UK, or APAC routing at account-creation. The matrix shows the GPU pool mapping per region and the failover behavior if the primary region is unavailable. EU customers stay in EU GPU pools; data never leaves the EU boundary.

6. EU AI Act Art. 14 Human-Oversight Operational Record

Per Art. 14, Replicate operators (i.e., the customer dev/ops team) must have effective oversight over the AI system. The operational record captures per-inference human-oversight events: model-pick validation, version-pick validation, input-validation step, output-review step. For high-risk systems (employment screening, education scoring, law-enforcement), Art. 14 imposes additional obligations that the binder addresses.

7. EU AI Act Art. 50 Transparency-Labeling

Per Art. 50, AI-generated outputs must be labeled as such. The transparency-labeling spec captures per-output provenance label: model-id + model-version + creation-timestamp + content-type. Customers integrating Replicate outputs into user-facing products pass this label through to the end-user UI per Art. 50's transparency obligations.

8. EU AI Act Art. 53(1)(b) GPAI Cascade

For models that are GPAI (general-purpose AI) under the Act — Llama 3, Mistral, Qwen, FLUX, SDXL — Art. 53(1)(b) requires training-data-summary disclosure. The cascade document covers all 100K models on Replicate: which are GPAI, which have published training-data summaries, which require downstream-cascade disclosure from the integrator.

9. GDPR Art. 28 Sub-Processor DPA Cascade
