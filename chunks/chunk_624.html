Why a single-browser audit binder does not cover MaxAI.me

Most ai_browser_extension audit binders target Chrome-only vendors. MaxAI ships the same extension to Chrome + Edge + Firefox + Safari under a single codebase, which multiplies the audit surface by 4:

  • Chrome MV3 — host_permissions, activeTab, storage, scripting, contextMenus, notifications. Chrome Web Store Data Safety disclosures.
  • Edge MV3 — same MV3 surface but Edge Add-ons Privacy + Microsoft Store privacy labeling each carry their own enforcement.
  • Firefox MV3 — Firefox's permission defaults are stricter than Chrome's; storage.partitioned + permissions kill-switch + Firefox AMO Data Disclosure.
  • Safari WebExtension — Safari App Store Nutrition Labels + Apple's privacy-manifest + per-purpose data-handling.

Four stores, four permission models, four disclosure formats, four enforcement bodies (FTC + EU + UK + Apple). A Chrome-only audit binder misses 75% of the surface.
