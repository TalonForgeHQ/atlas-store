  10. embedding_model_version — model behind similarity results.
  11. embedding_vector_set_hash — vector-store integrity receipt.

  15. evaluation_config_version — metric and threshold policy.
  16. evaluation_result_hash — scenario-to-sample diagnostics.
  17. human_promotion_approval_id — accountable release decision.
  18. cross_tenant_no_bleed_proof — isolation across labels, embeddings, models, and exports.

Gap 1 — dataset and sample lineage

Procurement needs source, license, transformation, deduplication, annotation, retention, and deletion lineage bound to each sample—not only a project-level dataset card.

Gap 2 — embeddings and similarity-search reproducibility

Every similarity result should cite the embedding model version, vector-set hash, filters, dataset version, and tenant boundary so a reviewer can reproduce why a sample was selected.

Gap 3 — annotation and plugin integrity

Human and automated label changes need actor identity, plugin version, input hash, output hash, review decision, and rollback evidence. Third-party extensions should not create an unaudited side channel.

Gap 4 — model-evaluation evidence

Scenario and sample-level diagnostics should join the evaluated model artifact, dataset version, slices, metrics, thresholds, failures, and human promotion decision in one exportable record.

Gap 5 — secure deployment and retention

Cloud, on-prem, and air-gapped deployments need equivalent access, retention, legal-hold, deletion, encryption, PII/PHI, and cross-tenant isolation evidence without relying on a cloud-only control plane.
