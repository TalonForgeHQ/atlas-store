2. The 5-audit-gap map — what an enterprise CIO needs to verify before any AI-data-security + DSPM + AI-Governance procurement in 2026

Both vendors (Cyera + Securiti) advertise a similar compliance posture (SOC 2 + ISO 27001 + GDPR + EU AI Act). The question for the 2026 enterprise buyer is not the surface claim but the audit-trail evidence behind it. Here are the 5 audit-trail gaps that any vendor-DD should map to the per-AI-decision-event join-table:

Audit-gap #1 — Per-AI-Model-Registry-registration + per-AI-data-asset-classification join-table (16-column)

When a customer registers a foundation-model (OpenAI GPT-4o + Anthropic Claude 4.5 + Google Gemini 2.5 + Meta Llama 4 + Mistral Large 2 + Cohere Command R+) or a fine-tuned model into the AI Model Registry, and the DSPM layer classifies the per-data-asset-uuid + per-AI-training-corpus + per-AI-fine-tune-corpus + per-AI-RAG-corpus, the vendor must produce a per-model-registration-uuid → per-AI-model-name + per-AI-model-version-pinned + per-AI-model-provider + per-AI-model-deployment-region + per-AI-model-fine-tune-corpus-hash + per-AI-model-RAG-corpus-hash + per-AI-model-inference-prompt-store-hash + per-AI-model-training-data-summary + per-AI-model-license-spdx-id + per-tenant-isolation-key + per-cross-tenant-AI-model-isolation-evidence + per-human-oversight-approval join-table that walks from a single registration back to model provider + deployment region + fine-tune corpus + tenant. Securiti's privacy@securiti.ai + dpo@securiti.ai is the canonical GDPR DPA + SOC 2 + ISO 27001 + ISO 42001 + AI Governance inquiry channel for this evidence (verified 2026-07-15 via securiti.ai/privacy-policy/). Cyera: privacy@cyera.io.

Audit-gap #2 — Per-AI-Training-Data-Lineage + per-AI-RAG-Corpus-Data-Classification + per-AI-Fine-Tune-Data-Provenance join-table (12-column)

For every AI model the customer registers, the AI Governance + DSPM modules trace per-AI-training-corpus + per-AI-fine-tune-corpus + per-AI-RAG-corpus + per-AI-inference-prompt-store. The vendor must trace per-corpus-uuid → per-corpus-source-uuid → per-corpus-license-spdx-id → per-corpus-license-version → per-corpus-data-classification-label (PII | PHI | PCI | confidential | internal | public) → per-corpus-cross-border-transfer-sccs-version-US-EU-UK-Switzerland for license-provenance + GDPR-DPA + EU-AI-Act-Annex-IV-§1(c)-training-data-provenance + Art. 53(d) GPAI training-data transparency + ISO 42001 6.2 evidence.

Audit-gap #3 — Per-AI-Inference-Data-Flow-Mapping + per-AI-Prompt-PII-Detection + per-AI-Output-Sanitization join-table (10-column)

When a customer AI agent issues an inference call (LLM API call + embedding API call + vector-search API call) and the AI Governance module performs per-inference-prompt-PII-detection + per-inference-output-PII-detection + per-inference-output-sanitization, the vendor must produce a per-inference-event-uuid → per-inference-prompt-hash → per-inference-completion-hash → per-inference-PII-detection-classification → per-inference-output-sanitization-decision → per-inference-OWASP-LLM-Top-10-mapping (LLM01-LLM10) → per-inference-tenant-isolation-evidence → per-inference-SIEM-export-event audit chain. This is OWASP LLM Top 10 + ISO 42001 8.4 + EU AI Act Art. 14 human-oversight + SOC 2 CC7.2 evidence.

Audit-gap #4 — Per-Consent-Management + per-Data-Subject-Rights-Automation + per-Cookie-Compliance join-table (11-column)

For GDPR Art. 6 (lawful basis) + Art. 7 (consent) + Art. 15-22 (data subject rights) + CCPA/CPRA + LGPD + PIPL compliance, the vendor must produce per-user-consent-record-uuid + per-DSR-event-uuid + per-cookie-consent-event-uuid + per-purpose-of-processing + per-data-category + per-recipient + per-cross-border-transfer-SCCs-version + per-tenant-isolation-key + per-AI-decision-event-linkage + per-quarterly-consent-audit-pass-event + per-DSR-fulfillment-SLA-event.
