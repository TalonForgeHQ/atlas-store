4. The Air-Gap Question — Tabnine's Unique Position

Tabnine is the only AI-code-assistant vendor in the canonical 6-vendor cohort that ships a true air-gapped Kubernetes deployment with zero egress. This means: no model telemetry leaves the cluster, no completion logging leaves the cluster, no anonymized analytics leaves the cluster, no license-check ping leaves the cluster, no LLM-provider-routed-call leaves the cluster (when the customer supplies their own model weights).

For buyers subject to FedRAMP Moderate / High, Defense Federal Acquisition Regulation Supplement (DFARS), International Traffic in Arms Regulations (ITAR), or financial-services-banking air-gap policy, Tabnine's air-gap posture answers the procurement question directly: "Show me the egress-firewall config that proves no outbound network from this instance under any condition, including license-check + telemetry + model-routing." Tabnine's per-air-gapped-deployment-flag is enforced at the egress-firewall layer (iptables + Kubernetes NetworkPolicy + Cilium), not just at the application layer.

Question to ask security@tabnine.com: "For an air-gapped deployment, what is the per-license-check protocol — is the cluster truly offline, or does it require a periodic outbound call for license validation? And how does the per-audit-log-export-id work without any outbound network?"

5. Zero-Retention-Default Since 2023 — Industry First

Tabnine made per-zero-retention-flag = true the default for all tiers (Free, Pro, Enterprise, Private Instance) in 2023 — 3 years before Cursor, Copilot, Cody, Continue, or Codeium shipped opt-in zero-retention modes. This is the strongest zero-retention posture in the cohort because:

  • Default-on, not opt-in: every Tabnine tenant is zero-retention by default. Competitor opt-in zero-retention requires the customer to remember to enable it.
  • Enforced at egress: Tabnine's zero-retention is enforced at the egress-firewall layer for hosted tiers and is the only behavior for on-prem + Private Instance + air-gapped tiers.
  • Verifiable in audit log: the per-zero-retention-flag is logged in every per-audit-log-export-id row, so procurement teams can reconcile the receipt themselves.
  • Per-tenant opt-out is impossible: Tabnine does not retain customer code or completions even if the customer wanted them to. There is no per-tenant opt-in to retention.

Question to ask security@tabnine.com: "Is the per-zero-retention-flag = true behavior verifiable from inside the cluster without any external network call? Specifically, can we export the per-audit-log-export-id to our S3 / Splunk / Datadog endpoint and see zero-retention-flag = true for every row, even for tenants created before 2023?"

  2. Does per-zero-retention-flag = true (default-on since 2023) apply to all tiers including Free + Pro, or only Enterprise + Private Instance?
  3. Can we export the per-audit-log-export-id to our own S3 / Splunk / Datadog endpoint with no Tabnine-side retention, even from inside the air-gapped cluster?
  4. Does per-BYO-LLM-api-key-version support rotation without affecting other tenants on the same Tabnine instance?
  5. What per-data-residency-region options exist — US-only / EU-only / apac-only / customer-VPC-only / fully-air-gapped?
  6. Is per-customer-code-isolation-region enforced at the index-namespace level, or only at the retrieval-API level?
  7. Does Tabnine ship a per-prompt-injection-detection-signal-id when a fetched code chunk tries to override the system prompt, and does that signal flow into the per-action-rollback-reason field for Tabnine Agents?
  8. What is the retention window for the per-AI-action-id ledger for Tabnine Agents actions, and is it configurable per-customer-tenant-id?
  9. Does per-action-rollback-id persist beyond the Tabnine Agents session, and for how long?
  10. Can we deploy Tabnine on-prem in an air-gapped Kubernetes cluster (per-air-gapped-deployment-flag = true) with no outbound network, including no license-check + no telemetry + no model-routing egress?
  11. Does per-VPC-peering-id support AWS PrivateLink / Azure Private Link / GCP Private Service Connect?
  12. Does Tabnine ship a per-SSO-SAML-OIDC-config-id with built-in Okta / Entra ID / Google Workspace templates, and does it work in air-gapped mode?
  13. Does SCIM provisioning (per-SCIM-provisioning-id) support group-based access control for the Tabnine Agents plugin?
