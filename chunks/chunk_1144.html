5-WEDGE non-overlap rubric (FIRST-PARTY verbatim 2026-07-24, PITFALL #99)

  1. Vault-Less SaaS with Distributed Fragments Cryptography (DFC): no master key, no single point of failure, sub-10ms retrieval. k-of-n threshold scheme with per-fragment per-replica per-region hash chain. Retrievable per-customer without DFC re-derivation. Distinct from HashiCorp Vault self-deployment and from cloud-only KMS (AWS KMS, GCP KMS, Azure Key Vault).
  2. AI Agent Identity & Dynamic Secrets ledger: per-agent spiffe-id + per-invocation ephemeral token + per-secret-TTL bucket + per-rotation cadence cross-tenant-no-bleed invariant. The only cohort member that binds human dev workload and non-human agent workload and runtime infrastructure (Lambda / Cloud Run / ECS / EKS / GKE / AKS) under the same per-request audit-log replay.
  3. MCP-aware workload-identity binding: per-agent-tool-call per-secret-decision audit-log replay with per-MCP-server auth boundary + per-invocation policy enforcement + per-decision signing. Pre-MCP cohorts only saw human dev + service-account workload identity; the agent-era cohort requires per-tool-call provenance.
  4. BYOK + HSM-less FIPS-140-3 + roll-your-own-KMS: bring-your-own-key across AWS KMS + GCP KMS + Azure Key Vault + HashiCorp Vault (where Vault is the integration endpoint, not the source-of-truth). FIPS-140-3 boundary without forcing customers to deploy HSM hardware. Distinct from cloud-only-KMS vendors and from open-source-only (Infisical + Doppler) without the managed-secret plane.
  5. Zero-Trust just-in-time access for human + non-human identities: per-request policy enforcement + per-session audit-log + per-customer retrievable receipt. The 2026 cohort is the first to require both human + non-human identity in the same per-request audit-log replay — without forcing customers to deploy HashiCorp Vault themselves.
