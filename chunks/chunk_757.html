Five buyer-facing joins the Replicate 757 audit scores

  1. Per-model version pinning across 100K+ models. Replicate publishes a per-model version hash on the replicate.com/models registry. The audit asks: what is the canonical evidence shape a Cursor or Notion security reviewer can hand to procurement to prove that SDXL-base-1.0 and SDXL-base-1.1 are pinned to separate cog containers with separate training-data-cutoff, license, and license-redistribution provenance?
  2. Per-cog container isolation proof. Each model on Replicate runs in an isolated Docker cog container with no shared memory or filesystem between concurrent inference requests. The audit asks Replicate to publish the isolation proof shape (memory, network, filesystem, IPC, syscalls) so the buyer can answer "is an adversarial input on Stable Video Diffusion leaking into the FLUX.1 [pro] container next door?"
  3. GPU-pool sub-processor cascade. Replicate runs across AWS GPU fleet + GCP GPU fleet + CoreWeave GPU fleet. The audit asks: what is the sub-processor DPA cascade, what is the per-region data residency logic, and what is the cross-region failover lineage when a GPU pool is exhausted?
  4. Per-inference deletion-cascade SLA. Replicate caches input + output + prompt + result + log per inference. The audit asks: what is the deletion-cascade SLA (input purge + output purge + cache purge + log purge + provable-purge-log) so a buyer can demonstrate GDPR Art. 17 (right-to-erasure) or HIPAA §164.524 (right-of-access)?
  5. EU AI Act Art. 50 + Art. 14 + Art. 53(1)(b) cascade across 100K+ models. Replicate is the canonical GPAI-hosting platform: the audit asks Replicate to publish per-model training-data-summary disclosure, per-output transparency-labeling provenance, and per-inference human-oversight event capture across the whole 100K+ catalog.
