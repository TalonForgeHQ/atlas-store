1. The 30 audit-gaps (grouped by evidence surface)

1.1 Per-agent-session-id + per-agent-run-id + per-trace-id + per-span-id core-trace-attestation (6 audit-gaps)

  1. Per-agent-session-id + per-agent-session-start-timestamp + per-agent-session-end-timestamp + per-agent-session-duration-ms — every AI-agent session must have a unique session-id + start/end timestamps + duration. The audit trail needs session-id + session-start + session-end + session-duration-ms for SOC 2 CC7.2 + EU AI Act Art. 12 traceability.
  2. Per-agent-run-id + per-agent-run-attempt-number + per-agent-run-parent-session-id + per-agent-run-fork-source — multi-turn agent runs need run-id + attempt-number + parent-session-id + fork-source for re-tracing session lineage. The audit trail needs the run-id + parent-session-id + fork-source.
  3. Per-trace-id (OpenTelemetry trace-id + W3C traceparent header) + per-trace-root-span-name + per-trace-root-span-kind (agent / chain / tool / llm / retrieval) — every agent invocation must produce an OpenTelemetry-compliant trace-id + a root-span-name + a root-span-kind. Helicone, Langfuse, Arize Phoenix, Honeyhive, Traceloop, LangSmith all emit OpenTelemetry-compatible trace-ids.
  4. Per-span-id + per-span-parent-span-id + per-span-name + per-span-kind (llm / tool / retriever / embedding / chain / agent / parser / retry / fallback) — every nested operation within an agent trace needs a span-id + parent-span-id + name + kind. The audit trail needs the span tree for the auditor to reconstruct the full agent execution graph.
  5. Per-span-start-timestamp + per-span-end-timestamp + per-span-duration-ms + per-span-status-code (ok / error / timeout / cancelled / rate-limited) — every span needs timing + status evidence for performance-baseline + SLO + incident-replay. SOC 2 CC7.3 (monitoring) + ISO 42001 §9.4 require this.
  6. Per-span-attribute-set (JSON-key-value-pairs) + per-span-event-set (timestamped-span-lifecycle-events) — span-attributes + span-events carry the AI-specific lineage (model-id, prompt-template-version, tool-call-id, retrieval-query, embedding-vector-id, tool-output). The audit trail needs both attributes + events for full reproducibility.

1.2 Per-LLM-model-id + per-LLM-prompt-hash + per-token-usage + per-cost-attribution LLM-call-attestation (6 audit-gaps)

  1. Per-LLM-model-id (claude-sonnet-4-5 / claude-opus-4 / gpt-4o / gpt-4-turbo / gemini-2.0-flash / llama-3.3-70b / mistral-large / veryfi-custom-fine-tuned / fixie-ultravox-realtime) + per-LLM-model-version-id + per-LLM-model-fine-tuning-run-id — every LLM call within an agent trace needs the model-id + model-version + fine-tuning-run-id. EU AI Act Art. 9 (risk management) + Art. 11 (technical documentation) + Art. 12 (logging) require this granularity.
  2. Per-LLM-prompt-hash (SHA-256 of prompt-template-rendered-content) + per-LLM-prompt-template-id + per-LLM-prompt-template-version-id — every LLM call needs a cryptographic prompt-hash + template-id + template-version-id so the auditor can reproduce the exact prompt that produced the output.
  3. Per-LLM-prompt-temperature + per-LLM-prompt-max-tokens + per-LLM-prompt-top-p + per-LLM-prompt-frequency-penalty + per-LLM-prompt-presence-penalty — every LLM call needs the sampling-parameters for reproducibility. The audit trail needs temperature + max-tokens + top-p + frequency-penalty + presence-penalty.
  4. Per-LLM-prompt-input-tokens + per-LLM-prompt-output-tokens + per-LLM-prompt-cache-hit-tokens + per-LLM-prompt-cache-miss-tokens — every LLM call needs token-usage breakdown including cache-hit + cache-miss for cost-attribution + provider-cache-evidence (Anthropic prompt-caching, OpenAI prompt-caching, Google context-caching).
  5. Per-LLM-call-cost-usd + per-LLM-call-cost-provider-id + per-LLM-call-cost-tier-id + per-LLM-call-cost-currency — every LLM call needs the cost + provider + tier + currency for cost-attribution + chargeback + finance-team reconciliation. The audit trail needs cost-usd + provider-id + tier-id + currency.
  6. Per-LLM-call-latency-ms (time-to-first-token + total-latency + inter-token-latency-avg) + per-LLM-call-streaming-flag — every LLM call needs latency breakdown for performance-baseline + SLO. The audit trail needs TTFT + total-latency + inter-token-latency + streaming-flag.

1.3 Per-tool-call-id + per-tool-name + per-tool-input + per-tool-output + per-tool-error tool-call-attestation (6 audit-gaps)

  1. Per-tool-call-id + per-tool-name (search_web / send_email / create_ticket / query_database / update_crm / run_query / generate_image / transcribe_audio) + per-tool-call-source (agent / chain / human-in-loop) — every tool call within an agent trace needs the call-id + name + source for SOC 2 CC7.2 + EU AI Act Art. 12 (logging of automated actions) + Art. 14 (human oversight).
  2. Per-tool-input-payload (JSON-serialized-input + input-hash) + per-tool-input-pii-redaction-tier (none / partial / full) — every tool call needs the input + input-hash + PII-redaction-tier for GDPR Art. 5 (data minimization) + Art. 32 (security of processing) + Schrems II.
  3. Per-tool-output-payload (JSON-serialized-output + output-hash) + per-tool-output-pii-redaction-tier + per-tool-output-truncation-flag — every tool call needs the output + output-hash + PII-redaction-tier + truncation-flag. The audit trail needs output-hash for the auditor to verify the output has not been retroactively modified.
  4. Per-tool-error-code (HTTP-4xx / HTTP-5xx / timeout / rate-limited / schema-validation-failed / auth-failed / quota-exceeded) + per-tool-error-message + per-tool-error-stack-trace-redacted — every tool call error needs error-code + message + redacted-stack-trace for incident-replay + SOC 2 CC7.4 (incident response).
  5. Per-tool-call-latency-ms + per-tool-call-retry-count + per-tool-call-retry-backoff-ms + per-tool-call-circuit-breaker-state (closed/open/half-open) — every tool call needs latency + retry-count + backoff + circuit-breaker-state for reliability-baseline.
  6. Per-tool-call-rollback-id + per-tool-call-rollback-reason + per-tool-call-rollback-timestamp + per-tool-call-rollback-status — every tool call that triggered a rollback (e.g. a database-update that was later reversed) needs rollback-id + reason + timestamp + status. This is the critical surface for EU AI Act Art. 14 (human oversight) + Art. 15 (accuracy, robustness, cybersecurity) + SOC 2 CC7.3 + CC7.4.

1.4 Per-retrieval-query-id + per-retrieval-source-id + per-retrieval-chunk-id + per-similarity-score retrieval-lineage (5 audit-gaps)

  1. Per-retrieval-query-id + per-retrieval-query-text + per-retrieval-query-embedding-model-id + per-retrieval-query-embedding-vector-id — every RAG retrieval call needs query-id + query-text + embedding-model-id + embedding-vector-id. The audit trail needs the embedding-model-id + vector-id for reproducibility (different embedding-models produce different similarity-scores).
  2. Per-retrieval-source-id (vector-db-id + collection-id + index-id) + per-retrieval-source-type (vector / BM25 / hybrid / SQL / GraphQL / API) — every retrieval call needs the source-id + source-type for SOC 2 CC6.1 (logical access) + GDPR Art. 28 (processor obligations on third-party-vector-db-vendors like Pinecone + Weaviate + Qdrant + Chroma).
  3. Per-retrieval-chunk-id + per-retrieval-chunk-text + per-retrieval-chunk-source-document-id + per-retrieval-chunk-source-page-id — every retrieved chunk needs chunk-id + chunk-text + source-document-id + source-page-id so the auditor can verify what context the LLM received.
  4. Per-similarity-score (cosine / dot-product / L2 / BM25 / hybrid-fusion-score) + per-rank-position + per-retrieval-filter-applied — every retrieved chunk needs the similarity-score + rank-position + filter-applied for ranking-reproducibility.
  5. Per-retrieval-prompt-injection-detection-signal-id + per-retrieval-source-poisoning-detection-flag + per-retrieval-metadata-filter-bypass-detection-flag — every retrieval call needs OWASP-LLM-Top-10-LLM06 (sensitive information disclosure) + LLM01 (prompt injection) + retrieval-source-poisoning + metadata-filter-bypass detection signals. EU AI Act Art. 9 (risk management) + Art. 15 (cybersecurity) require this.
