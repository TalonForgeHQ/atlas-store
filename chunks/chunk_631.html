Applied example — Voiceflow multi-agent collaboration handoff provenance

Voiceflow's multi-agent collaboration feature is the architectural decision point under EU AI Act Art. 14. In a single customer conversation, the user's question may be handed off between Agent A (intent classifier) → Agent B (FAQ retriever) → Agent C (transactional handler) → Agent D (human escalation approver) within Voiceflow itself. Under EU AI Act Art. 14, the audit trail for a multi-agent chain must capture:

  • Agent invocation — which agent invoked which + timestamp + which conversation ID + which user + which channel
  • Context handoff — what intermediate state was passed between agents (intent label + retrieved-doc-id + response draft + user-history)
  • Human-oversight event — which approver + which agent step + which accept/edit/reject decision + content-hash
  • Sub-processor invocation per agent — which LLM + which region + which retention window + which voice-AI sub-processor per agent call
  • Channel pivot — if the handoff crosses channel boundaries (Web Chat → Twilio Voice escalation), which channel pivot occurred with which retention + which wiretap-law trigger

This is not theoretical — it's the audit packet that closes EU AI Act Art. 14 obligations for Voiceflow's customer-experience agents in BFSI (transactional) + retail (purchase decision) + telco (account changes) + healthcare (triage).
