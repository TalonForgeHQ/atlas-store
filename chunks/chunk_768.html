Five buyer-facing joins

  1. Slite Agent self-maintenance provenance + write-back ledger: agent identity/version, workspace + scope + doc set, LLM sub-processor (OpenAI vs. Anthropic vs. self-hosted), proposed state-change set, human approval/override events, and the actual write-back receipts (which doc was renamed/merged/tagged/unverified).
  2. AI Search answer provenance across MCP-connected sources: user identity, question text, per-source documents/versions (Slack + Notion + Google Drive + GitHub + Linear + Jira + Salesforce per the Integrations catalog), LLM sub-processor invoked, retention window, region routing, and citation set attached to the answer.
  3. Cross-workspace data-isolation audit: workspace-A retrieval cannot read workspace-B documents; workspace-A vectors are not searchable from workspace-B; workspace-A prompt context cannot leak into workspace-B answer; workspace-B audit events cannot be triggered by workspace-A Agent runs. Where is the isolation boundary enforced (retrieval layer, embedding layer, LLM context layer, audit-log layer) and how is it tested?
  4. MCP server surface audit: per-tool OAuth scope required, per-tenant authorization honored, per-tool data classes read/written, per-tool downstream side effects possible, per-tool rate-limit + retention + region window. How does this surface interact with EU AI Act Art. 50 transparency-labeling (MCP tool outputs are AI-generated, but the prompt an MCP caller sends is not necessarily labeled)?
  5. Sub-processor cascade DPA + 14-day change-notification SLA: Slite AI Search + Agent retrieval pulls from MCP-connected sources (typically 5-10 first-party sub-processors per workspace), forwards the assembled prompt to OpenAI and Anthropic (2 LLM sub-processors), and serves the answer back through Slite web/CLI/MCP — totaling 12+ sub-processors in the per-answer cascade. DPA cascade + 14-day change-notification SLA per GDPR Art. 28(2) must enumerate the new Super 2025 sub-processors.

Common evidence failures

  • Showing a Slite Agent self-maintenance pass without the proposed state-change set, the human approval/override events, the LLM sub-processor invoked, and the write-back receipts (which doc was actually renamed/merged/tagged).
  • Logging an AI Search answer without the per-source documents/versions across MCP-connected sources, the LLM sub-processor invoked, the retention window, the region routing, and the citation set.
  • Recording a Slite Agent run that crossed workspace boundaries without the isolation boundary test (retrieval layer + embedding layer + LLM context layer + audit-log layer).
  • Promoting an MCP tool that reads/writes a sensitive data class without the OAuth scope, the per-tenant authorization, the rate-limit + retention + region window, and the EU AI Act Art. 50 transparency-labeling artifact.
  • Deleting a Slite workspace while omitting MCP-connected-source copies, Ask Slite retrieval indexes, AI Search caches, agent-run traces, evaluations, logs, replicas, and backups — or omitting the new Super 2025 sub-processors from the DPA cascade.
