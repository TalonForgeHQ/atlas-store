20-column AI agent audit export schema (replay-portable, 2026)

  1. tenant_id — first-party issued, KMS-pinned, never reused across tenants.
  2. ai_agent_id — per-agent stable identifier with version, lifecycle (provisioned → active → decommissioned), and human owner-of-record.
  3. ai_agent_invocation_id — per-run UUID, monotonic, exportable.
  4. prompt_hash — SHA-256 of the rendered prompt + system + tool-context, per tenant, salted with the tenant KMS key.
  5. completion_hash — SHA-256 of the streamed completion tokens, same salt.
  6. model_provider_id — e.g. openai-gpt-4o / anthropic-claude-opus-4 / google-vertex-gemini-2-5 / self-hosted-llama-3-3-70b.
  7. model_version_id — exact snapshot, not "latest."
  8. tool_call_id + tool_name + tool_decision (allow / deny / sandbox-preview / human-required) — per-tool audit row.
  9. mcp_server_id + mcp_tool_call_id + mcp_guardrail_decision — MCP-server surface separately auditable.
  10. rag_retrieval_id + vector_collection_id + vector_namespace_id + retrieved_chunk_id + similarity_score — RAG line of sight.
  11. ai_handoff_id + ai_handoff_reason + human_override_id + human_acknowledged_at — EU AI Act Art. 14(4) human-oversight continuity.
  12. data_residency_aws_region + data_residency_proof — first-party sub-processor registry pin.
  13. cross_tenant_no_bleed_invariant — boolean + cryptographic proof hash (zero-knowledge or signed-merkle-root).
  14. audit_export_id + audit_export_format (jsonl / parquet / csv / signed-cbor) + audit_export_signed_at + audit_export_signing_key_id — the export itself is signed.
  15. replay_hash — deterministic hash of the entire row that reproduces byte-identical when the same inputs are re-run.
  16. retention_until + deletion_propagation_id — GDPR Art. 17 / CCPA / LGPD deletion path.
  17. sub_processor_id + sub_processor_change_notification_sla — DORA Article 28 sub-processor lineage.
  18. soc2_type2_report_id + eu_ai_act_article_id + nist_ai_rmf_function_id + iso_42001_clause_id — framework crosswalk at row level.

Five failure modes the audit-export reproducibility test will catch in 2026

  1. Hashing without a tenant salt. "We hash prompts" means SHA-256 of the literal bytes. Two tenants with the same prompt produce the same hash, and the auditor can correlate. Demand a per-tenant KMS salt.
  2. Tool-call decisions without the deny row. Allowed tool calls are logged; denied tool calls are dropped. The auditor cannot reconstruct what was prevented. Demand every decision (allow / deny / sandbox-preview / human-required).
  3. RAG without similarity score. The retrieval is logged but not the score. The auditor cannot reconstruct why a chunk was retrieved. Demand the similarity_score + threshold + reranker id.
  4. AI agent handoff without human_acknowledged_at. The handoff row exists, but the human never acknowledged. Demand the human_acknowledged_at timestamp + override id.
  5. Audit export signed but not replayable. The export is signed with a vendor-controlled key and the auditor cannot verify integrity without the vendor. Demand a public verification path or a customer-held signing key (BYOK).
