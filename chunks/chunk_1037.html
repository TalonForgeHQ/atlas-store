What an evidence-gap map for ai agent long-term memory needs to cover

An ai agent long-term memory platform persists context, persona, facts, and skills across sessions. Procurement needs more than a vector-store row. It needs an immutable receipt that joins every memory add, recall, update, and delete to the downstream LLM call and tool action that depended on it. The five cohort vendors — Mem0, Zep, Letta, Cognee, and Redis Long-Term Memory — each pick a different substrate for that plane. The map shows how to read those differences without locking into any one vendor's demo narrative.

Substrate comparison: memory-CRUD vs GraphRAG vs persona-state vs knowledge-graph vs vector-store-memory

Each vendor owns a distinct long-term-memory substrate. Procurement should map their workflow's data shape to the substrate, not the other way around.

  • Mem0 (mem0ai): memory-CRUD-first substrate. Apache-2.0 OSS repo github.com/mem0ai/mem0 with 12.6K+ stars. YC W24. Founders Taranjeet Singh (CEO) + Deshraj Yadav (CTO). Add + Search + Update + Delete four-named primitives. Semantic + episodic + procedural memory trio. Verbatim first-party integrations OpenAI + Anthropic + LangChain + LlamaIndex.
  • Zep (getzep.com): GraphRAG + auto-summarization substrate. YC W22. Founders Daniel Chalef (CEO) + Alex Reibman (CTO). Verbatim four buffer types: Message Window + Summary + Token + Perpetual. Fact extraction into structured fact table. Cohort-unique conversation-window-as-product surface.
  • Letta (letta.com): persona-state substrate with persistent agent memory. UC Berkeley Sky Lab origin. Founders Sarah Wooders + Charles Packer. Cohort-unique persona-state plane where every agent has a private long-term state schema distinct from a vector store.
  • Cognee (cognee.ai): knowledge-graph substrate that ingests conversations + documents and produces an entity-relationship graph. Open-source Apache-2.0. Cohort-unique EKG (episodic knowledge graph) plane distinct from a pure vector store.
  • Redis Long-Term Memory (redis.io): vector-store-memory substrate that extends Redis with long-term memory primitives on top of Redis Stack. Cohort-unique in-memory-first plane with the smallest persistence-floor but the lowest latency.

How the five substrates map to SOC 2 CC7.2 + EU AI Act Art. 12 + GDPR Art. 17

Every long-term-memory decision needs a verifiable chain. SOC 2 CC7.02 requires that system monitoring detect and act on policy violations. EU AI Act Art. 12 requires automatic logging of events over the agent's lifecycle. GDPR Art. 17 requires erasure on request with proof. The receipt's memory_delete_event_id, cross_user_no_bleed_audit_trail, and audit_export_id columns are the ones procurement should inspect first. None of the five vendors export all three out of the box; the gap map names which platform covers which column.

Why memory-first ai agent platforms are not interchangeable with eval/observability/contract layers

Memory is a write-and-recall plane. Eval/observability (Braintrust, Langfuse, Confident AI, Arize AI, LangSmith) is a read-and-trace plane. Contracts and authorization (Soda, Monte Carlo, C1, Cerbos) are a check-and-decide plane. If a vendor's "memory" is just a vector-store wrapper, the receipt is missing the episodic + procedural + fact-evidence columns. If a vendor's "memory" is just a write log, the receipt is missing the semantic-match and recall-ranking columns. The five-vendor benchmark in this cohort separates the memory-only vendors from the eval/observability vendors that bolt on memory primitives as a feature.
