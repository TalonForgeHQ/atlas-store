2. The 28-column join-table schema

The canonical 28-column Dataiku audit join-table:

  1. per-project-id — DSS project root
  2. per-flow-id — Dataiku Flow definition
  3. per-flow-version-id — Immutable Flow version (every recipe save = new version)
  4. per-recipe-id — Recipe inside a Flow
  5. per-recipe-version-id — Immutable Recipe version
  6. per-model-id — ML/AI model registered
  7. per-model-version-id — Immutable Model version
  8. per-model-artifact-id — Pickle / ONNX / serialized artifact with hash
  9. per-model-deployment-id — Live API endpoint / batch inference job
  10. per-model-deployment-version-id — Immutable Deployment version
  11. per-prediction-id — Real-time or batch prediction record
  12. per-batch-prediction-id — Batch-scoring row provenance
  13. per-feature-id — Feature Store feature
  14. per-feature-store-version-id — Immutable Feature Store version
  15. per-dataset-id — DSS dataset
  16. per-dataset-version-id — Immutable Dataset partition/version
  17. per-managed-folder-id — Managed folder (S3 / Azure / GCS-backed)
  18. per-managed-folder-version-id — Immutable folder content version
  19. per-model-evaluation-id — Per-deploy eval score (accuracy / AUC / RMSE / fairness)
  20. per-drift-detection-id — Per-monitor data + concept drift signal
  21. per-model-bias-audit-id — Dataiku Govern bias-audit record (per-protected-attribute + per-subgroup + per-fairness-metric)
  22. per-explainability-record-id — SHAP / LIME / Integrated-Gradients run tied to a prediction
  23. per-lineage-edge-id — Flow → Recipe → Dataset → Model provenance edge
  24. per-lineage-edge-version-id — Immutable lineage edge version
  25. per-tenant-id — Cloud SaaS tenant root
  26. per-billing-account-id — Billing / cost-attribution layer
  27. per-deployment-region — AWS / Azure / GCP region mapping (Schrems II + FedRAMP SC-7)
  28. per-immutable-hash — WORM hash for immutability verification
