Q2: What's the difference between sub-agent handoff provenance and per-agent-action provenance?

Per-agent-action provenance captures a single agent's actions (which user + which workspace + which agent + which prompt + which LLM sub-processor + which retention window + which region routing). Sub-agent handoff provenance captures the chain of agent invocations (which Agent A invoked which Agent B with which context + which intermediate state). For multi-agent systems (Taskade + Lindy + any AI Operating System), the EU AI Act Art. 14 audit-trail must span both: you need to know what Agent A did AND what Agent A told Agent B AND what Agent B did with that context. Lindy's drag-and-drop sub-agent handoff UI is the surface that makes the multi-agent audit-trail mandatory.
