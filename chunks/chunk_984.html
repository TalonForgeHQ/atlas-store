Why the agent-secret evidence chain matters

An agent can be governed at the identity layer and still create an audit gap if its credential material cannot be traced. Procurement teams need a replay that identifies the agent, workload, secret version, approval policy, injection target, tool call, rotation event, and revocation event.

Five evidence joins

  1. Agent identity to secret request: identify the requesting agent and the policy version evaluated.
  2. Secret version to workload injection: prove which version entered which runtime without exposing the value.
  3. Credential to tool call: correlate the credential lease with the downstream action it enabled.
  4. Certificate or key to lifecycle event: retain issuance, operation, expiry, rotation, and revocation timestamps.
  5. Privileged session to export: join approval, session, commands, tenant isolation, and immutable evidence export.
