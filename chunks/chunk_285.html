Audit Gap #1 — End-to-end provenance join-table

The SOC 2 CC7.2 + ISO 27701 A.7.34 + ISO 42001 9.4 + ISO 27001 A.8.34 + PCI DSS Req. 10 + HIPAA Security Rule + Schrems II + India-DPDP-Act-2023 + EU AI Act Art. 12 + Aug 2026 GPAI enforcement baseline for a modern AI-era data-catalog requires a 50+ column provenance join-table with at minimum:

  • per-Alation-data-asset-id → the row in Alation's catalog
  • per-Alation-data-classification-id → the classification (PII / PCI / HIPAA / GDPR-special-category / LGPD-sensivel / India-DPDP-personal-data / PIPL-China-personal-information)
  • per-Alation-AI-era-classification-id → which AI-era policy applies (LLM-call / AI-Agent / Agentic-AI-Workflow / MCP-server-call / tool-call / prompt-template-version)
  • per-Alation-LLM-call-id + per-Alation-AI-Agent-id + per-Alation-Agentic-AI-Workflow-id + per-Alation-MCP-server-call-id + per-Alation-tool-call-id + per-Alation-prompt-template-version-id + per-Alation-completion-id + per-Alation-token-id
  • per-Alation-knowledge-base-id + per-Alation-knowledge-base-document-id + per-Alation-knowledge-base-chunk-id + per-Alation-RAG-retrieval-id
  • per-Alation-Data-Lineage-id (with Collibra 458 it's per-Collibra-Data-Lineage-id; with Informatica 457 it's per-Informatica-Data-Lineage-id)
  • per-Alation-cloud-account-id + per-Alation-region-id + per-Alation-data-residency-id
  • per-Alation-tenant-id + per-Alation-workspace-id + per-Alation-vendor-id
  • per-Alation-DSAR-id + per-Alation-consent-id
  • per-Alation-ML-Model-id + per-Alation-ML-training-data-id
  • per-Alation-billing-event-id

Alation's native Data Lineage graph models the upstream → downstream flow at the column level, but the AI-era extension (LLM-call-id through billing-event-id) is not yet a first-class join — buyers today have to build it themselves on top of the catalog's open APIs. Collibra 458's Data Lineage has a similar shape (open-API-first, AI-era is extension-on-top); Informatica 457's CLAIRE AI engine surfaces some AI-era lineage out-of-the-box but at the MDM-tier pricing that excludes 80%+ of the 6000-customer mid-market.

Audit Gap #2 — OWASP LLM Top 10 + MITRE ATLAS coverage-matrix

The Alation-specific coverage-matrix for the OWASP LLM Top 10 LLM01+LLM02+LLM03+LLM06+LLM07+LLM08 + MITRE ATLAS + EU AI Act Art. 15 + NIST AI RMF MEASURE + ISO 42001 6.1.4 + Schrems II + India-DPDP-Act-2023 + Aug 2026 GPAI enforcement alignment requires 24+ columns covering:

  • Threat IDs: per-OWASP-LLM-Top-10-id + per-MITRE-ATLAS-id + per-attack-id
  • Attack shapes: per-prompt-injection-id + per-jailbreak-id + per-multi-turn-attack-id + per-multi-modal-attack-id
  • Knowledge-base attacks: per-RAG-retrieval-poisoning-id + per-knowledge-base-document-poisoning-id
  • Agentic-AI attacks: per-Agentic-AI-poisoning-id + per-MCP-server-call-poisoning-id + per-tool-call-poisoning-id
  • ML-model attacks: per-ML-Model-poisoning-id + per-ML-training-data-poisoning-id
  • Catalog-specific attacks: per-Data-Lineage-poisoning-id + per-Data-Catalog-poisoning-id + per-Active-Metadata-poisoning-id (Alation-specific — Collibra 458 has per-Data-Catalog-poisoning-id + per-Data-Lineage-poisoning-id but not Active-Metadata; Informatica 457 has per-CLAIRE-AI-recommendation-poisoning-id + per-Data-Lineage-poisoning-id + per-Data-Catalog-poisoning-id)
  • Exfil attacks: per-data-exfiltration-via-LLM-id + per-shadow-AI-data-leakage-id

Across a 6000-customer Alation install base, this coverage-matrix is the single most-asked-about artifact in the EU AI Act Art. 15 readiness workshops. Most buyers today build it as a spreadsheet mapping their Alation catalog rows to MITRE ATLAS techniques by hand — and the AI-era rows (LLM01-LLM08) are blank because the catalog doesn't yet expose the threat-axis as a first-class column.

Audit Gap #3 — Per-defense-row catalogue

For each threat row in Gap #2, the auditor needs a defense-row mapping. The Alation-specific defense catalogue requires 20+ columns per row covering prompt-injection-defense + jailbreak-defense + multi-turn-attack-defense + data-poisoning-defense + MCP-server-call-poisoning-defense + tool-call-poisoning-defense + RAG-retrieval-poisoning-defense + knowledge-base-document-poisoning-defense + Agentic-AI-poisoning-defense + ML-Model-poisoning-defense + ML-training-data-poisoning-defense + Active-Metadata-validation + Data-Lineage-validation + Data-Catalog-validation + customer-PII-redaction-defense + PCI-card-data-redaction-defense + HIPAA-PHI-redaction-defense + human-supervision-defense. Alation ships some of these as default behavior (PII redaction at classification-time, lineage validation at sync-time) but the AI-era defenses (prompt-injection, jailbreak, MCP-server-call-poisoning, Agentic-AI-poisoning) are extension-on-top. Collibra 458 ships similar defaults at the catalog-level with AI-era as extension-on-top. Informatica 457 has the strongest out-of-the-box AI-era defenses thanks to CLAIRE AI but at MDM-tier pricing.

Vendor-selection rubric (the buyer's answer)

If your 2026 priority is AI-era audit-trail (Gap #1 + #2 + #3): Informatica 457 wins on out-of-the-box AI-era coverage (CLAIRE AI engine) but at MDM-tier pricing. If your priority is data-catalog UX + Active Metadata: Alation 459 wins on the breadth of the 6000-customer install base + the Connected Sheets + ML Feature Store surface. If your priority is EU public-sector + DoD FedRAMP-High: Collibra 458 wins on FedRAMP Moderate + DoD IL5 readiness (Alation has FedRAMP Moderate only). Most enterprise buyers run a two-vendor stack: Alation for catalog + Connected Sheets + Active Metadata, Informatica for MDM + CLAIRE AI, Collibra for public-sector — and the audit-gap join-table is built across all three.
