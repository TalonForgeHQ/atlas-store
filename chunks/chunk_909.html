Protect AI MLSecOps Audit Map: AVID + ModelScan + AIBOM + LLM Guard + Radar Evidence Replay

This page is a buyer-side evidence map for Protect AI's MLSecOps platform. It targets the long-tail questions enterprise security + GRC + MLOps + AI governance buyers ask after a vendor demo: can one AI vulnerability be replayed from public AVID disclosure, through the affected model artifact + AIBOM node, to the LLM Guard + Radar + Spliceguard control that surfaces or contains it — across customer tenants with isolation proof?

Short answer

Protect AI positions itself as the first-mover MLSecOps platform with the open-source AI Vulnerability Database (AVID) community as the public baseline, plus ModelScan for static ML-model security scanning, AI Bill of Materials (AIBOM) for the model-supply-chain lineage, Radar for AI asset discovery, LLM Guard for runtime prompt + output sanitization, Spliceguard for ML model-poisoning detection, and GenAI Security Posture Management as the platform's control surface. The Palo Alto Networks acquisition closed mid-2024; the Protect AI brand and engineering leadership continue operating.

The practical procurement question is not whether the platform has the longest feature list. It is whether one AVID-disclosed AI vulnerability — say a model-format deserialization flaw or a prompt-injection chain — can be pinned to the affected model artifact + version + AIBOM node, routed through the ModelScan finding + LLM Guard decision + Radar discovery event, and produced as a single exportable machine-readable evidence chain for an external auditor, across all customer tenants using the model.

Protect AI's first-party buyer-visible wedge

  • Open-source AVID community baseline: github.com/protectai/ai-vulnerability-database is the public registry any enterprise can query before engaging Protect AI as a vendor — only cohort member with this open-source verification baseline.
  • ModelScan open-source: github.com/protectai/modelscan scans TensorFlow Lite, PyTorch, ONNX, Keras, SavedModel, Pickle, joblib, OpenAI GPT-2 weights and more — only cohort member with an open-source static-ML-model scanner.
  • AIBOM substrate: AI Bill of Materials pattern matches the CycloneDX SBOM standard — cohort-unique machine-readable artifact for the model supply chain.
  • LLM Guard: github.com/protectai/llm-guard open-source runtime prompt + output sanitization layer — cohort-unique because cohort peers (Mindgard, HiddenLayer, Pillar) ship equivalent surfaces commercially, not open-source.
  • Spliceguard: ML model-poisoning detection layer — cohort-unique wedge.
  • Radar AI asset discovery: cloud-account-aware shadow-AI + AI asset inventory — cohort-unique among the four siblings.
  • Palo Alto Networks distribution: post-acquisition enterprise distribution footprint (Prisma AIRS rebrand option) — deepest enterprise-distribution moat of cohort.

Buyer-side cold-file questions target: can a Fortune-500 reviewer receive one machine-readable export that joins the public AVID disclosure to the model artifact + AIBOM node + LLM Guard runtime decision + Radar discovery event for every customer tenant that has deployed the affected model?

The 5 buyer-facing audit replay questions

  1. AVID-to-tenant join: for every AI/ML vulnerability in the AVID open-source database, can Protect AI enumerate exactly which customers + which model artifacts + which AIBOM nodes have been affected?
  2. ModelScan reproducibility: across scanner version + threshold + ruleset + supplier-ML-model-version, are findings reproducible or do they shift between repeat scans?
  3. LLM Guard end-to-end replay: for every prompt + tool + retrieval-context combination that triggered a block, can the original input + LLM Guard decision + downstream inference call be replayed deterministically?
  4. Radar discovery lineage: cloud-account + IAM role + data-source + lineage depth — can a breach review be scoped in minutes instead of hours via single-dashboard export?
  5. Unified export join: posture findings + red-team events + runtime decisions + AVID disclosures — joined as one machine-readable export without manual reconciliation across four separate JSONs?
