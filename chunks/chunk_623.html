Why AI browser extensions need a different SOC 2 evidence binder

Classical SOC 2 Type II letters cover server-side SaaS controls. AI browser extensions break that model because:

  • Manifest V3 permission surface — host_permissions, activeTab, storage, scripting. Every permission grant is a data-flow declaration that flows into GDPR + Chrome Web Store Data Safety.
  • Per-tab-query provenance — every LLM call is triggered by a URL + page content snapshot. The audit trail is per-tab, not per-API-call.
  • Multi-LLM sub-processor chain — 6 LLM sub-processors (GPT-4 + Claude + Gemini + Llama + Mistral + DeepSeek) under one extension. Each carries its own DPA, retention window, training-data opt-out, region.
  • Consumer + prosumer + enterprise deployer mix — 1M+ installs across all three personas. SOC 2 evidence must serve enterprise procurement, Chrome Web Store Data Safety must serve consumer trust.
  • EU AI Act Art. 14 human-oversight — must be captured per-account per-session, not per-tenant per-day.

The 12-document AI-plane binder (the audit deliverable)

  1. AI Control Matrix (ACM) — 24-column provenance join-table mapping every Chrome permission grant + every LLM sub-processor + every page-content snapshot to a SOC 2 CC control + an EU AI Act article + a GDPR article.
  2. Manifest V3 Permission-Posture Diagram — host_permissions (which domains), activeTab (transient per-tab access), storage (which keys), scripting (which injected scripts). Per-permission per-region per-LLM-sub-processor.
  3. Per-Tab-Query Provenance Record — URL + page-content hash + LLM sub-processor choice + retention window + region routing + training-data opt-out flag, captured per Merlin invocation.
  4. Multi-LLM Sub-Processor Audit Trail — every LLM call (which model, which jurisdiction, which data residency, which retention window, which training-data opt-out flag). 6 LLMs = 6 sub-processor DPAs.
  5. Citation Provenance Record (LLM-output ↔ webpage-source) — when Merlin summarizes a YouTube video, every claim in the summary traces back to a transcript timestamp. When Merlin answers a Gmail reply, every paragraph traces back to a quoted email fragment.
  6. Deletion Cascade Runbook — account-closed → 30-day retention → provable purge log across Merlin gateway + all 6 LLM sub-processors.
  7. Sub-Processor Map (current + change-notification cadence) — OpenAI + Anthropic + Google + Meta + Mistral + DeepSeek + at least one secondary vendor. Each with their own sub-processor disclosure + change-notification mechanism + audit rights.
  8. Prompt-Injection Runbook (webpage-poisoning defenses) — LLM01 mitigation patterns when an attacker injects malicious DOM content that Merlin reads as instructions.
  9. Human-Oversight Log Sample (Art. 14 evidence) — per-account per-session log of Merlin-response-accepted / rejected / edited / regenerated events. Provable human-in-the-loop evidence.
  10. Hallucination Boundary Evidence — when Merlin is asked about a URL it can't read, what does it say? Documented failure mode + user-facing disclosure.
  11. Region-Pinning Attestation — EU + UK + US + Canadian + Australian users see different data-residency guarantees. Per-region attestation per-LLM-sub-processor.
  12. EU AI Act Gap Memo (Aug 2 2026 enforcement) — Art. 14 + Art. 50 + Art. 53(1)(b) + Art. 53(d) per-cohort applicability + per-Merlin-feature applicability.

A: Because the extension reads every page you visit, every email you draft, every PDF you open. That data-flow surface is broader than most enterprise SaaS. CISOs at Series B+ + Fortune-500 require SOC 2 evidence before procurement.

Q2: What's the highest-risk audit gap?
A: Manifest V3 permission-posture documentation. Chrome Web Store Data Safety disclosures are enforceable by the FTC + EU regulators but rarely audited per-permission per-region per-LLM-sub-processor. The audit wedge.

Q3: How is this different from a server-side AI vendor?
A: Server-side AI vendors control their own infrastructure. Browser-extension AI vendors depend on Chrome's permission model + 6 LLM sub-processors + per-page content snapshots. The audit surface is wider and the controls are split across more parties.
