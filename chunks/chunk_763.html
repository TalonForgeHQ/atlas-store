Five buyer-facing joins

  1. Identity → ontology → AIP model/Skill: user/session, project role, AIP model, Skill version, instructions/system prompt, ontology object/path, branch, tenant, account, and region.
  2. AIP Run → tool action → ontology/policy decision: AIP Run ID, message/tool call, code/LLM action, AIP Function call, branch/tag action, workshop/code-workbook write; identity, authorization, AIP policy decision (allow/deny/limit/branch-guard), input/output, credential boundary, ontology object, and downstream state change.
  3. Branch/version/merge → decision audit: ontology branch/version/merge, AIP Run/tool calls, AIP policy decisions, reviewer, commit, audit log entry, and rollback target.
  4. Feedback → evaluation → release: end-user rating/comment, evaluation dataset/version/metric/reviewer, AIP model/Skill/instruction change, Foundry ontology revision, promoted release, rollback target, and release timestamp.
  5. Export → deletion: ontology objects, AIP Runs, messages, tool inputs/outputs, code artifacts, audit logs, feedback, evaluations, caches, replicas, backups, region, subprocessor, completion time, and exception state.

Common evidence failures

  • Showing an AIP final answer without the AIP Run, the message/tool-call sequence, the AIP Function invoked, the ontology object touched, and the AIP policy decision recorded.
  • Logging an AIP Function or workshop write without the project role, authorization decision, AIP policy decision, credential boundary, ontology object, and downstream state hash.
  • Reporting an ontology branch merge without the reviewer, AIP Run/tool calls bound to that branch, audit log entry, and rollback target.
  • Promoting an AIP model or Skill revision without the evaluation dataset/version/metric, reviewer, ontology revision tag, and rollback target.
  • Deleting an AIP Run or Thread while omitting messages, tool inputs/outputs, code artifacts, audit logs, feedback, evaluation copies, caches, replicas, or backups.
