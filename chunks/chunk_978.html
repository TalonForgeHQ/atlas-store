AI Identity Governance Vendor Benchmark: 5 Access Evidence Gaps

Published July 23, 2026 · Atlas Audit Prep · long-tail keyword: AI identity governance vendor benchmark

SailPoint, Okta, CyberArk, Saviynt, and Microsoft Entra ID Governance approach identity security from different foundations. A useful comparison tests whether each platform can replay a human, workload, non-human, privileged, or AI-agent access decision—not whether five feature lists use the same labels.

Research basis: this benchmark consolidates five completed vendor-specific evidence maps in the ai_identity_governance cohort. It uses public product and trust surfaces, does not claim access to private controls, and does not assert that any vendor fails a control.

Gap 1: identity-to-owner lineage

Can an auditor trace one identity from source, sponsor, credential, workload, or agent registration through lifecycle state, requested resource, policy version, entitlement, and final revocation using stable identifiers?

Gap 2: decision and approval chain

Does each request preserve the recommendation, policy and model version, segregation-of-duties result, approver, justification, ticket, conditional-access result, and final grant or denial without requiring screenshot reconstruction?

Gap 3: privileged and agent action boundary

Can the export distinguish eligible privilege, just-in-time activation, AI-generated recommendation, autonomous agent action, human approval, emergency override, rollback, and post-action review?

Gap 4: failure-path and revocation retention

Denied, expired, retried, cancelled, orphaned, dormant, overridden, and rolled-back access states should retain the same actor, reason, timestamp, tenant, region, and policy lineage as successful decisions.

Gap 5: regulatory evidence export

Can the record support SOC 2, ISO/IEC 27001, ISO/IEC 42001, NIST AI RMF, EU AI Act Article 14 human oversight, NIS2 incident reconstruction, and SEC cyber disclosure evidence requests in a reproducible export?

Minimum replay schema

  • tenant_id, identity_id, identity type, sponsor, source system, and regional boundary;
  • credential or token ID, requested resource, entitlement, policy ID/version, and model version;
  • recommendation, SoD result, approver, justification, ticket, activation, and action IDs;
  • override, rollback, expiry, revocation, reviewer, failure reason, and final-state timestamps;
  • control ID/version, evidence SHA-256, exporter identity, retention state, and export hash.

How to run the benchmark

  1. Select one representative human, workload, privileged, or AI-agent access journey per platform.
  2. Request the complete replay package, including denied, expired, overridden, and revoked states.
  3. Normalize all five packages into the minimum schema above.
  4. Score each required join red, amber, or green.
  5. Document which missing joins still require screenshots, support tickets, or manual testimony.

What identity and compliance teams should receive

The output should be a five-row comparison, the highest-priority missing receipts, one sample identity-to-decision-to-action replay, and a Trust Center or procurement publication checklist. The goal is a reproducible evidence map rather than a generic vendor ranking.
