25-question AI agent vendor RFI template (2026)

  1. Legal name + LEI + HQ + VAT + managing directors. First-party imprint, not a Crunchbase page.
  2. Latest audited financials (10-K / annual report / Companies House). First-party PDF, not a "Series D" headline.
  3. ISO/IEC 27001:2022 certificate + certifying body + audit date + expiry. First-party certificate PDF.
  4. ISO/IEC 27701:2019 certificate (PIMS) — first-party.
  5. ISO/IEC 42001:2023 AIMS certificate — first-party.
  6. SOC 2 Type II report — first-party PDF (Type II, not Type I).
  7. SOC 2 Type II observation period dates — at least 6 months.
  8. HIPAA attestation (if applicable) — first-party BAA template + per-customer BAA tracker.
  9. GDPR DPA template — first-party.
  10. DORA Compliance Package — first-party, with audit + exit + step-in rights + TLPT cooperation.
  11. DORA Addendum — signed for EU financial-sector buyers.
  12. EU AI Act readiness evidence per Article (Art. 6 + Art. 9 + Art. 10 + Art. 13 + Art. 14 + Art. 15 + Art. 26 + Art. 27 + Art. 50 + Annex III high-risk list).
  13. NIST AI RMF GOVERN/MAP/MEASURE/MANAGE coverage with per-citation evidence-pinning.
  14. Data residency: per-tenant pinning to AWS region(s) with first-party sub-processor registry.
  15. Sub-processor change-notification SLA (e.g., 30 days) and per-sub-processor data-residency evidence.
  16. AI agent identity governance: per-agent + per-run + per-tenant identifier scheme.
  17. AI agent authorization: per-call authorization + per-tool-call allow/deny list + per-MCP-server scoping.
  18. AI agent handoff to human: per-handoff receipt with reason + human override id + acknowledgment timestamp.
  19. Prompt + completion hashing per tenant to prevent cross-tenant bleed and to make audit exports deterministic.
  20. Tool-call + MCP-server + guardrail decision audit trail with per-call latency, cost, status.
  21. Vector memory isolation per tenant with per-collection + per-namespace + per-tenant KMS key.
  22. Cross-tenant no-bleed invariant — demonstrable on every replay (no shared keys, no shared tokens, no shared audit log path).
  23. Audit export id + replay artifact — deterministic, machine-readable, replayable.
  24. SLA evidence: 24×7 monitoring + incident-notice SLA + RTO/RPO + postmortem cadence.
  25. Last 3 customer-reported security incidents in the last 24 months — first-party postmortem URLs, not buried marketing copy.

Five failure modes the RFI will surface in 2026

  1. "SOC 2 Type II" with a Type I report. A Type I is a point-in-time attestation; Type II requires an observation period. The vendor will hand you a Type I and call it a Type II. Demand the observation-period dates.
  2. "ISO 42001 certified" with no certificate. "In progress" + "self-attested" + "alignment" are not certificates. Demand the first-party certificate PDF + certifying body + audit date.
  3. "DORA aligned" without a signed DORA Addendum. A marketing page is not a signed addendum. Demand the first-party signed addendum with audit + exit + step-in rights + TLPT cooperation.
  4. "EU-hosted" with US telemetry breach. The app servers sit in Frankfurt, but the analytics + product-improvement telemetry leaves the EU. Sub-processor cascade review is the only way to detect this.
  5. "AI agent" with no human-in-the-loop receipt. The agent takes high-risk actions autonomously with no `ai_handoff_id` + `ai_handoff_reason` + `human_override_id`. The audit will treat it as fully autonomous and most EU AI Act-aligned buyers will reject it.
