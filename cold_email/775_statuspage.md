## Distinct CLOSER wedge

This closes the cohort's status-page-centric gap. incident.io covers all-in-one response and AI Scribe/Workflows/Insights; Rootly covers AI-RCA confidence, IDE remediation, and PagerDuty migration; Better Stack covers MCP + AI SRE + a broad observability substrate; FireHydrant covers AI Enriched Incidents, Zoom/Google Meet live video transcription, and PagerDuty/Opsgenie migration. Statuspage adds customer-facing communication control: which audience sees which component state, which template is rendered, which channel is delivered, and which subscriber receipt proves the update.

## Five audit joins

1. Incident/alert → approved update → selected page type/audience → rendered status-page snapshot → notification receipt.
2. Public/private/audience-specific page isolation → component scope → subscriber list → recipient/channel targeting.
3. Template/version → author and approver → timestamp → rendered message → update-history/export receipt.
4. Third-party component source → freshness/status input → mapped component → downstream page state → rollback receipt.
5. Jira/Confluence/REST API/integration action → actor or service identity → human approval where applicable → customer-visible output → retention/deletion/export receipt.
